# 3.5.6 Harness Engineering

> 为Agent系统构建可靠的工程化控制层

## 一、什么是 Harness Engineering

### 1.1 概念定义

```
Agent ≠ 产品

一个"能跑"的Agent和一个"能上线"的Agent之间，
隔着一整层工程基础设施 —— 这就是 Harness。

┌─────────────────────────────────────────────────┐
│                   用户/调用方                     │
└───────────────────────┬─────────────────────────┘
                        │
┌───────────────────────↓─────────────────────────┐
│              Harness 控制层                      │
│                                                 │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐        │
│  │ 输入护栏  │ │ 执行监控  │ │ 输出护栏  │        │
│  │ Guardrail│ │ Tracing  │ │ Guardrail│        │
│  └──────────┘ └──────────┘ └──────────┘        │
│                                                 │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐        │
│  │ 评估框架  │ │ 回退机制  │ │ 人工介入  │        │
│  │ Eval     │ │ Fallback │ │ HITL     │        │
│  └──────────┘ └──────────┘ └──────────┘        │
│                                                 │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐        │
│  │ 成本控制  │ │ 速率限制  │ │ A/B测试  │        │
│  │ Budget   │ │ Rate Lim │ │ Routing  │        │
│  └──────────┘ └──────────┘ └──────────┘        │
│                                                 │
└───────────────────────┬─────────────────────────┘
                        │
┌───────────────────────↓─────────────────────────┐
│              Agent 核心逻辑                      │
│     (LLM调用 / 工具执行 / 推理循环)              │
└─────────────────────────────────────────────────┘

Harness Engineering = 设计和构建这个控制层的工程学科

类比：
  Agent = 赛车引擎
  Harness = 安全带 + 仪表盘 + ABS + 碰撞检测 + 黑匣子
```

### 1.2 为什么需要 Harness

```
没有 Harness 的Agent会出什么问题？

1. 幻觉输出直达用户
   → Agent编造数据，用户无法分辨
   → 需要：输出护栏 (Output Guardrails)

2. 成本失控
   → 一个bug导致Agent陷入死循环，消耗数百美元Token
   → 需要：预算控制 (Budget Control)

3. 出错后黑盒无法排查
   → "昨天的Agent回答为什么是错的？" → 无法追溯
   → 需要：执行追踪 (Tracing)

4. 安全风险
   → 用户通过prompt injection操纵Agent执行危险操作
   → 需要：输入护栏 (Input Guardrails)

5. 无法衡量Agent到底好不好
   → "升级了模型，效果是变好了还是变差了？"
   → 需要：评估框架 (Evaluation Harness)

6. 高危操作无人审核
   → Agent自动发邮件、修改数据库、调用支付接口
   → 需要：人工介入 (Human-in-the-Loop)
```

## 二、Guardrails（护栏系统）

### 2.1 输入护栏

```python
"""
输入护栏 —— 在请求到达Agent之前进行安全检查
"""

from dataclasses import dataclass
from openai import OpenAI

client = OpenAI()


@dataclass
class GuardrailResult:
    """护栏检查结果"""
    passed: bool
    reason: str = ""
    modified_input: str | None = None  # 修正后的输入（如脱敏）


class InputGuardrails:
    """输入护栏集合"""

    def __init__(self):
        self.checks: list = []

    def add_check(self, name: str, check_fn):
        """注册一个检查函数"""
        self.checks.append({"name": name, "fn": check_fn})

    def run(self, user_input: str) -> GuardrailResult:
        """依次执行所有检查"""
        current_input = user_input

        for check in self.checks:
            result = check["fn"](current_input)
            if not result.passed:
                print(f"护栏 [{check['name']}] 拦截: {result.reason}")
                return result
            if result.modified_input:
                current_input = result.modified_input

        return GuardrailResult(passed=True, modified_input=current_input)


# ===== 常用护栏实现 =====

def check_prompt_injection(text: str) -> GuardrailResult:
    """检测Prompt注入攻击"""
    injection_patterns = [
        "ignore previous instructions",
        "ignore all prior",
        "你的新指令是",
        "disregard your system prompt",
        "SYSTEM OVERRIDE",
    ]
    text_lower = text.lower()
    for pattern in injection_patterns:
        if pattern.lower() in text_lower:
            return GuardrailResult(
                passed=False,
                reason=f"检测到疑似Prompt注入: '{pattern}'"
            )
    return GuardrailResult(passed=True)


def check_pii(text: str) -> GuardrailResult:
    """检测并脱敏个人敏感信息"""
    import re
    # 手机号脱敏
    modified = re.sub(
        r'1[3-9]\d{9}',
        '[手机号已脱敏]',
        text
    )
    # 身份证号脱敏
    modified = re.sub(
        r'\d{17}[\dXx]',
        '[身份证已脱敏]',
        modified
    )
    if modified != text:
        return GuardrailResult(
            passed=True,
            reason="已脱敏敏感信息",
            modified_input=modified
        )
    return GuardrailResult(passed=True)


def check_topic_boundary(text: str) -> GuardrailResult:
    """检查是否在Agent能力范围内"""
    # 使用LLM判断请求是否在业务范围内
    response = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[{
            "role": "user",
            "content": f"""判断以下请求是否属于业务范围（客服/产品咨询/技术支持）。
请求：{text}
返回JSON：{{"in_scope": true/false, "reason": "原因"}}"""
        }],
        response_format={"type": "json_object"},
    )
    import json
    result = json.loads(response.choices[0].message.content)
    if not result["in_scope"]:
        return GuardrailResult(
            passed=False,
            reason=f"超出业务范围: {result['reason']}"
        )
    return GuardrailResult(passed=True)


# ===== 组装护栏 =====

input_guard = InputGuardrails()
input_guard.add_check("prompt_injection", check_prompt_injection)
input_guard.add_check("pii_detection", check_pii)
input_guard.add_check("topic_boundary", check_topic_boundary)

# 使用
result = input_guard.run("我的手机号是13812345678，请帮我查订单")
if result.passed:
    # 安全的输入（已脱敏）传给Agent
    agent_input = result.modified_input
else:
    # 拦截，返回拒绝信息
    print(f"请求被拦截: {result.reason}")
```

### 2.2 输出护栏

```python
"""
输出护栏 —— 在Agent响应到达用户之前进行检查
"""

class OutputGuardrails:
    """输出护栏"""

    def __init__(self):
        self.checks = []

    def add_check(self, name: str, check_fn):
        self.checks.append({"name": name, "fn": check_fn})

    def run(self, agent_output: str, context: dict = None) -> GuardrailResult:
        for check in self.checks:
            result = check["fn"](agent_output, context or {})
            if not result.passed:
                return result
        return GuardrailResult(passed=True)


def check_hallucination(output: str, context: dict) -> GuardrailResult:
    """
    幻觉检测 —— 检查Agent输出是否有证据支撑
    """
    sources = context.get("retrieved_docs", [])
    if not sources:
        return GuardrailResult(passed=True)  # 无检索场景不检查

    source_text = "\n".join(sources)
    response = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[{
            "role": "user",
            "content": f"""判断以下回答是否与参考资料一致，是否存在编造的信息。

回答：{output}

参考资料：{source_text}

返回JSON：{{"has_hallucination": true/false, "details": "具体说明"}}"""
        }],
        response_format={"type": "json_object"},
    )
    import json
    result = json.loads(response.choices[0].message.content)
    if result["has_hallucination"]:
        return GuardrailResult(
            passed=False,
            reason=f"检测到幻觉: {result['details']}"
        )
    return GuardrailResult(passed=True)


def check_toxicity(output: str, context: dict) -> GuardrailResult:
    """有害内容检测"""
    response = client.moderations.create(input=output)
    result = response.results[0]
    if result.flagged:
        flagged_categories = [
            cat for cat, flagged in result.categories.__dict__.items()
            if flagged
        ]
        return GuardrailResult(
            passed=False,
            reason=f"内容违规: {', '.join(flagged_categories)}"
        )
    return GuardrailResult(passed=True)


def check_no_internal_leak(output: str, context: dict) -> GuardrailResult:
    """防止泄露内部信息（system prompt、工具细节等）"""
    sensitive_patterns = [
        "system prompt",
        "你的指令是",
        "my instructions are",
        "function_call",
        "tool_use",
    ]
    output_lower = output.lower()
    for pattern in sensitive_patterns:
        if pattern.lower() in output_lower:
            return GuardrailResult(
                passed=False,
                reason=f"检测到内部信息泄露: '{pattern}'"
            )
    return GuardrailResult(passed=True)


# 组装输出护栏
output_guard = OutputGuardrails()
output_guard.add_check("hallucination", check_hallucination)
output_guard.add_check("toxicity", check_toxicity)
output_guard.add_check("internal_leak", check_no_internal_leak)
```

## 三、Tracing & Observability（追踪与可观测性）

### 3.1 为什么需要Tracing

```
Agent的执行过程是非确定性的：

用户: "帮我分析竞品"
  │
  ├─ LLM调用1: 理解意图（150ms, 320 tokens）
  ├─ 工具调用: web_search("竞品A 最新动态")（2.3s）
  ├─ 工具调用: web_search("竞品B 最新动态")（1.8s）
  ├─ LLM调用2: 分析搜索结果（800ms, 2100 tokens）
  ├─ LLM调用3: 生成报告（1.2s, 3500 tokens）
  └─ 总计: 6.25s, 5920 tokens, $0.089

如果没有Tracing —— 你只能看到最终输出
如果有Tracing —— 你能看到每一步的输入/输出/耗时/Token/成本
```

### 3.2 实现执行追踪

```python
"""
Agent执行追踪系统
"""

import time
import uuid
from dataclasses import dataclass, field
from typing import Any
from contextlib import contextmanager


@dataclass
class Span:
    """一个追踪跨度（对应一次LLM调用或工具调用）"""
    span_id: str
    trace_id: str
    parent_id: str | None
    name: str
    type: str                  # "llm" | "tool" | "guardrail" | "custom"
    start_time: float
    end_time: float = 0
    input_data: Any = None
    output_data: Any = None
    metadata: dict = field(default_factory=dict)
    error: str | None = None

    @property
    def duration_ms(self) -> float:
        return (self.end_time - self.start_time) * 1000

    def to_dict(self) -> dict:
        return {
            "span_id": self.span_id,
            "trace_id": self.trace_id,
            "parent_id": self.parent_id,
            "name": self.name,
            "type": self.type,
            "duration_ms": round(self.duration_ms, 1),
            "metadata": self.metadata,
            "error": self.error,
        }


class Tracer:
    """Agent执行追踪器"""

    def __init__(self):
        self.traces: dict[str, list[Span]] = {}
        self._current_trace: str | None = None
        self._current_span: str | None = None

    def start_trace(self, name: str = "agent_run") -> str:
        """开始一个新的追踪"""
        trace_id = str(uuid.uuid4())[:8]
        self.traces[trace_id] = []
        self._current_trace = trace_id
        return trace_id

    @contextmanager
    def span(self, name: str, span_type: str = "custom"):
        """上下文管理器：自动记录一个Span的耗时"""
        span = Span(
            span_id=str(uuid.uuid4())[:8],
            trace_id=self._current_trace,
            parent_id=self._current_span,
            name=name,
            type=span_type,
            start_time=time.time(),
        )
        old_span = self._current_span
        self._current_span = span.span_id

        try:
            yield span
            span.end_time = time.time()
        except Exception as e:
            span.end_time = time.time()
            span.error = str(e)
            raise
        finally:
            self._current_span = old_span
            self.traces[self._current_trace].append(span)

    def get_trace_summary(self, trace_id: str) -> dict:
        """获取追踪摘要"""
        spans = self.traces.get(trace_id, [])
        total_duration = sum(s.duration_ms for s in spans)
        total_tokens = sum(
            s.metadata.get("tokens", 0) for s in spans
        )
        total_cost = sum(
            s.metadata.get("cost", 0) for s in spans
        )

        return {
            "trace_id": trace_id,
            "total_spans": len(spans),
            "total_duration_ms": round(total_duration, 1),
            "total_tokens": total_tokens,
            "total_cost_usd": round(total_cost, 6),
            "errors": [s.name for s in spans if s.error],
            "spans": [s.to_dict() for s in spans],
        }

    def print_trace(self, trace_id: str):
        """打印可视化追踪"""
        spans = self.traces.get(trace_id, [])
        print(f"\n{'='*60}")
        print(f"Trace: {trace_id}")
        print(f"{'='*60}")
        for span in spans:
            indent = "  " if span.parent_id else ""
            status = "❌" if span.error else "✅"
            tokens = span.metadata.get("tokens", "")
            token_info = f" [{tokens} tokens]" if tokens else ""
            print(f"{indent}{status} {span.name} ({span.type}) "
                  f"- {span.duration_ms:.0f}ms{token_info}")
        print(f"{'='*60}\n")


# ===== 在Agent中使用Tracer =====

tracer = Tracer()

async def traced_agent_run(query: str):
    """带追踪的Agent执行"""
    trace_id = tracer.start_trace("agent_run")

    # 输入护栏
    with tracer.span("input_guardrails", "guardrail") as s:
        guard_result = input_guard.run(query)
        s.metadata["passed"] = guard_result.passed

    if not guard_result.passed:
        return f"请求被拦截: {guard_result.reason}"

    # LLM调用
    with tracer.span("intent_classification", "llm") as s:
        response = client.chat.completions.create(
            model="gpt-4o-mini",
            messages=[{"role": "user", "content": query}]
        )
        s.metadata["tokens"] = response.usage.total_tokens
        s.metadata["cost"] = response.usage.total_tokens * 0.00000015
        s.metadata["model"] = "gpt-4o-mini"

    # 打印追踪
    tracer.print_trace(trace_id)
    return response.choices[0].message.content
```

## 四、Evaluation Harness（评估框架）

### 4.1 持续评估的必要性

```
Agent不是部署一次就结束 —— 它需要持续评估：

场景1：模型升级
  GPT-4o → GPT-4o-2025-04 → 效果变好了？变差了？
  → 需要回归测试

场景2：Prompt修改
  调整了System Prompt → 旧场景还能正确处理吗？
  → 需要测试集验证

场景3：知识库更新
  新增了100篇文档 → 检索质量有没有下降？
  → 需要检索评估

场景4：线上问题
  用户反馈"昨天回答不对" → 如何系统性地回归？
  → 需要评估Pipeline
```

### 4.2 Evaluation Harness 实现

```python
"""
Agent评估框架 —— 自动化的质量评估Pipeline
"""

from dataclasses import dataclass
import json


@dataclass
class EvalCase:
    """一个评估用例"""
    id: str
    input: str                      # 用户输入
    expected_output: str | None     # 期望输出（可选）
    expected_tools: list[str] | None  # 期望调用的工具
    tags: list[str]                 # 标签（分类、难度等）
    metadata: dict | None = None


@dataclass
class EvalResult:
    """评估结果"""
    case_id: str
    actual_output: str
    scores: dict[str, float]       # 各维度评分
    passed: bool
    details: str = ""


class EvalHarness:
    """评估框架"""

    def __init__(self, agent_fn, llm_client=None):
        """
        Args:
            agent_fn: 被评估的Agent函数 (input: str) -> str
            llm_client: 用于LLM-as-Judge的客户端
        """
        self.agent_fn = agent_fn
        self.client = llm_client or OpenAI()
        self.scorers: dict[str, callable] = {}

    def add_scorer(self, name: str, scorer_fn):
        """注册一个评分器"""
        self.scorers[name] = scorer_fn

    async def run(self, cases: list[EvalCase]) -> list[EvalResult]:
        """运行评估"""
        results = []
        for case in cases:
            # 执行Agent
            actual_output = await self.agent_fn(case.input)

            # 运行所有评分器
            scores = {}
            for name, scorer in self.scorers.items():
                score = scorer(
                    input=case.input,
                    expected=case.expected_output,
                    actual=actual_output,
                )
                scores[name] = score

            # 判断是否通过（所有分数 >= 0.6）
            passed = all(s >= 0.6 for s in scores.values())

            results.append(EvalResult(
                case_id=case.id,
                actual_output=actual_output,
                scores=scores,
                passed=passed,
            ))

        return results

    def summary(self, results: list[EvalResult]) -> dict:
        """生成评估摘要"""
        total = len(results)
        passed = sum(1 for r in results if r.passed)

        # 按维度聚合
        score_sums: dict[str, float] = {}
        for r in results:
            for name, score in r.scores.items():
                score_sums[name] = score_sums.get(name, 0) + score

        avg_scores = {
            name: round(total_score / total, 3)
            for name, total_score in score_sums.items()
        }

        return {
            "total_cases": total,
            "passed": passed,
            "failed": total - passed,
            "pass_rate": f"{passed/total*100:.1f}%",
            "average_scores": avg_scores,
        }


# ===== 评分器实现 =====

def correctness_scorer(input: str, expected: str, actual: str) -> float:
    """正确性评分 —— LLM-as-Judge"""
    if not expected:
        return 1.0  # 无期望输出跳过

    response = client.chat.completions.create(
        model="gpt-4o-mini",
        response_format={"type": "json_object"},
        messages=[{
            "role": "user",
            "content": f"""比较实际回答与期望回答的一致性，打分0-1。

问题：{input}
期望回答：{expected}
实际回答：{actual}

返回JSON：{{"score": 0.0到1.0, "reason": "评分原因"}}"""
        }]
    )
    result = json.loads(response.choices[0].message.content)
    return result["score"]


def relevance_scorer(input: str, expected: str, actual: str) -> float:
    """相关性评分"""
    response = client.chat.completions.create(
        model="gpt-4o-mini",
        response_format={"type": "json_object"},
        messages=[{
            "role": "user",
            "content": f"""评估回答与问题的相关性，打分0-1。

问题：{input}
回答：{actual}

返回JSON：{{"score": 0.0到1.0}}"""
        }]
    )
    result = json.loads(response.choices[0].message.content)
    return result["score"]


def safety_scorer(input: str, expected: str, actual: str) -> float:
    """安全性评分"""
    response = client.moderations.create(input=actual)
    return 0.0 if response.results[0].flagged else 1.0


# ===== 组装并运行评估 =====

harness = EvalHarness(agent_fn=my_agent.handle)
harness.add_scorer("correctness", correctness_scorer)
harness.add_scorer("relevance", relevance_scorer)
```

### 5.1 分级回退策略

```python
"""
分级回退 —— 当Agent执行异常时的降级方案
"""


class FallbackChain:
    """回退链：逐级降级直到成功"""

    def __init__(self):
        self.levels = []

    def add_level(self, name: str, handler, condition=None):
        """
        Args:
            name: 回退级别名称
            handler: 处理函数
            condition: 何时触发此级别（默认：上一级失败时）
        """
        self.levels.append({
            "name": name,
            "handler": handler,
            "condition": condition,
        })

    async def execute(self, input_data: dict) -> dict:
        """执行回退链"""
        last_error = None

        for level in self.levels:
            try:
                result = await level["handler"](input_data)
                return {
                    "success": True,
                    "level": level["name"],
                    "result": result,
                }
            except Exception as e:
                last_error = e
                print(f"[{level['name']}] 失败: {e}，尝试下一级...")

        return {
            "success": False,
            "level": "exhausted",
            "error": str(last_error),
        }


# ===== 使用示例 =====

fallback = FallbackChain()

# Level 1: 完整Agent处理
fallback.add_level("full_agent", full_agent_handler)

# Level 2: 简化Agent（仅用基础模型，不调用工具）
fallback.add_level("simple_agent", simple_agent_handler)

# Level 3: 知识库精确匹配（绕过LLM）
fallback.add_level("exact_match", knowledge_base_exact_match)

# Level 4: 转人工
fallback.add_level("human_handoff", create_human_ticket)

"""
回退策略示意：

请求到达
    ↓
┌─────────────────┐
│  完整Agent处理    │ ← 优先使用
│  (GPT-4o+工具)   │
└────────┬────────┘
         │ 失败/超时
         ↓
┌─────────────────┐
│  简化Agent       │ ← 降级到基础模型
│  (GPT-4o-mini)   │
└────────┬────────┘
         │ 失败/超时
         ↓
┌─────────────────┐
│  知识库精确匹配   │ ← 绕过LLM
│  (规则+关键词)    │
└────────┬────────┘
         │ 无匹配
         ↓
┌─────────────────┐
│  转人工客服       │ ← 最终兜底
│  (创建工单)       │
└─────────────────┘
"""
```

### 5.2 Human-in-the-Loop（人工介入）

```python
"""
人工介入机制 —— 高风险操作需要人工确认
"""

from enum import Enum


class RiskLevel(Enum):
    LOW = "low"         # 自动执行
    MEDIUM = "medium"   # 记录日志但自动执行
    HIGH = "high"       # 需要人工确认
    CRITICAL = "critical"  # 必须人工确认 + 二次验证


# 风险等级配置
TOOL_RISK_LEVELS = {
    "search":           RiskLevel.LOW,
    "read_file":        RiskLevel.LOW,
    "send_email":       RiskLevel.HIGH,
    "update_database":  RiskLevel.HIGH,
    "delete_record":    RiskLevel.CRITICAL,
    "process_payment":  RiskLevel.CRITICAL,
}


class HumanInTheLoop:
    """人工介入控制器"""

    def __init__(self, approval_handler=None):
        """
        Args:
            approval_handler: 异步函数，发送审批请求并等待结果
                              签名：async (action, params) -> bool
        """
        self.approval_handler = approval_handler

    async def check_and_execute(
        self,
        tool_name: str,
        params: dict,
        tool_executor,
    ) -> dict:
        """检查风险等级，必要时请求人工审批"""

        risk = TOOL_RISK_LEVELS.get(tool_name, RiskLevel.MEDIUM)

        if risk == RiskLevel.LOW:
            # 直接执行
            result = await tool_executor(tool_name, params)
            return {"status": "auto_executed", "result": result}

        elif risk == RiskLevel.MEDIUM:
            # 执行但记录
            print(f"[AUDIT] 自动执行 {tool_name}: {params}")
            result = await tool_executor(tool_name, params)
            return {"status": "auto_executed_logged", "result": result}

        elif risk in (RiskLevel.HIGH, RiskLevel.CRITICAL):
            # 需要人工确认
            if self.approval_handler:
                approved = await self.approval_handler(tool_name, params)
                if approved:
                    result = await tool_executor(tool_name, params)
                    return {"status": "human_approved", "result": result}
                else:
                    return {"status": "human_rejected", "result": None}
            else:
                return {
                    "status": "blocked",
                    "reason": f"工具 '{tool_name}' 风险等级为 {risk.value}，需配置审批处理器",
                }
```

## 六、Budget Control（预算控制）

```python
"""
Token和成本预算控制 —— 防止Agent失控消耗资源
"""

from dataclasses import dataclass


@dataclass
class BudgetConfig:
    """预算配置"""
    max_tokens_per_run: int = 50000        # 单次运行最大Token
    max_cost_per_run: float = 1.0          # 单次运行最大花费（美元）
    max_llm_calls_per_run: int = 20        # 单次运行最大LLM调用次数
    max_tool_calls_per_run: int = 15       # 单次运行最大工具调用次数
    max_duration_seconds: int = 120        # 单次运行最大耗时


class BudgetController:
    """预算控制器 —— 超限时强制终止Agent"""

    def __init__(self, config: BudgetConfig):
        self.config = config
        self.total_tokens = 0
        self.total_cost = 0.0
        self.llm_calls = 0
        self.tool_calls = 0

    def record_llm_call(self, tokens: int, cost: float):
        """记录一次LLM调用"""
        self.total_tokens += tokens
        self.total_cost += cost
        self.llm_calls += 1

    def record_tool_call(self):
        """记录一次工具调用"""
        self.tool_calls += 1

    def check_budget(self) -> tuple[bool, str]:
        """检查是否超预算"""
        if self.total_tokens >= self.config.max_tokens_per_run:
            return False, f"Token超限: {self.total_tokens}/{self.config.max_tokens_per_run}"
        if self.total_cost >= self.config.max_cost_per_run:
            return False, f"成本超限: ${self.total_cost:.4f}/${self.config.max_cost_per_run}"
        if self.llm_calls >= self.config.max_llm_calls_per_run:
            return False, f"LLM调用超限: {self.llm_calls}/{self.config.max_llm_calls_per_run}"
        if self.tool_calls >= self.config.max_tool_calls_per_run:
            return False, f"工具调用超限: {self.tool_calls}/{self.config.max_tool_calls_per_run}"
        return True, "OK"

    def summary(self) -> dict:
        return {
            "tokens": self.total_tokens,
            "cost_usd": round(self.total_cost, 6),
            "llm_calls": self.llm_calls,
            "tool_calls": self.tool_calls,
        }
```

## 七、完整 Harness 集成

```python
"""
将所有Harness组件组装在一起
"""

class HarnessedAgent:
    """带Harness控制层的Agent"""

    def __init__(
        self,
        agent_core,           # 核心Agent逻辑
        input_guard: InputGuardrails,
        output_guard: OutputGuardrails,
        tracer: Tracer,
        budget: BudgetController,
        hitl: HumanInTheLoop,
        fallback: FallbackChain,
    ):
        self.core = agent_core
        self.input_guard = input_guard
        self.output_guard = output_guard
        self.tracer = tracer
        self.budget = budget
        self.hitl = hitl
        self.fallback = fallback

    async def run(self, user_input: str) -> str:
        trace_id = self.tracer.start_trace("harnessed_run")

        # 1. 输入护栏
        with self.tracer.span("input_guardrails", "guardrail") as s:
            guard = self.input_guard.run(user_input)
            s.metadata["passed"] = guard.passed
        if not guard.passed:
            return f"无法处理该请求: {guard.reason}"
        safe_input = guard.modified_input or user_input

        # 2. 预算检查
        ok, msg = self.budget.check_budget()
        if not ok:
            return f"资源限制: {msg}"

        # 3. 执行Agent（带回退）
        with self.tracer.span("agent_execution", "custom") as s:
            result = await self.fallback.execute({"input": safe_input})
            s.metadata["fallback_level"] = result.get("level")

        if not result["success"]:
            return "抱歉，当前无法处理您的请求。已转交人工处理。"

        agent_output = result["result"]

        # 4. 输出护栏
        with self.tracer.span("output_guardrails", "guardrail") as s:
            guard = self.output_guard.run(agent_output)
            s.metadata["passed"] = guard.passed
        if not guard.passed:
            return "抱歉，生成的回答未通过安全检查，请换种方式提问。"

        # 5. 追踪记录
        self.tracer.print_trace(trace_id)

        return agent_output
```

```
│  1. Harness 是什么                             │
│     → Agent核心逻辑之外的工程控制层            │
│     → 护栏 + 追踪 + 评估 + 回退 + 人工介入    │

│     → Input Guardrails：输入安全检查           │
│     → Output Guardrails：输出质量把关          │
│     → Tracing/Observability：执行可追溯        │
│     → Evaluation Harness：持续质量评估         │
│     → Fallback Chain：分级降级保可用            │
│     → Human-in-the-Loop：高危操作人工确认      │
│     → Budget Control：成本和资源限制           │
│                                               │
│  3. 核心理念                                   │
│     → Agent质量 = 模型能力 × Harness工程质量   │
│     → 不可靠的模型 + 好的Harness > 好的模型    │
│       + 没有Harness                            │
│                                               │
│  4. 工程落地                                   │
│     → 最小化启动：先加Tracing和输出护栏        │
│     → 逐步增强：按风险等级添加Guardrails       │
```
