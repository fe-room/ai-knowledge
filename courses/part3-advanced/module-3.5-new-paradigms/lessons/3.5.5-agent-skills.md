# 3.5.5 Agent Skills 体系

> 从硬编码能力到可组合、可发现、可学习的技能架构

## 一、什么是 Agent Skills

### 1.1 从"工具"到"技能"的演进

```
传统工具（Tool）：
  → 一个函数：输入参数 → 执行 → 返回结果
  → 无上下文：不知道何时该用、用多少次
  → 无组合：工具之间互相独立

技能（Skill）：
  → 一个完整的能力单元：知道何时触发、如何执行、如何验证
  → 有上下文：携带前置条件、后置检查、失败回退
  → 可组合：多个Skill可以编排成更复杂的Skill

类比：
  Tool = 一把螺丝刀
  Skill = "拆装电脑主板"（知道用什么螺丝刀、什么顺序、哪些注意事项）
```

### 1.2 Skill 的结构

```
┌─────────────────────────────────────────────────┐
│                    Skill                         │
├─────────────────────────────────────────────────┤
│                                                 │
│  名称：data_analysis                            │
│  描述：分析结构化数据并生成可视化报告            │
│                                                 │
│  ┌────────────────────────────────────┐         │
│  │ 前置条件 (Preconditions)           │         │
│  │ - 存在可读取的数据文件              │         │
│  │ - 数据格式为CSV/JSON/Excel         │         │
│  └────────────────────────────────────┘         │
│                                                 │
│  ┌────────────────────────────────────┐         │
│  │ 执行步骤 (Steps)                   │         │
│  │ 1. 加载数据 → tool: read_file      │         │
│  │ 2. 清洗数据 → tool: code_exec      │         │
│  │ 3. 统计分析 → tool: code_exec      │         │
│  │ 4. 生成图表 → tool: code_exec      │         │
│  │ 5. 撰写报告 → llm: generate_text   │         │
│  └────────────────────────────────────┘         │
│                                                 │
│  ┌────────────────────────────────────┐         │
│  │ 后置验证 (Postconditions)          │         │
│  │ - 报告包含至少一个数据表格          │         │
│  │ - 报告包含至少一个可视化图表        │         │
│  │ - 结论有数据支撑                    │         │
│  └────────────────────────────────────┘         │
│                                                 │
│  ┌────────────────────────────────────┐         │
│  │ 失败回退 (Fallback)               │         │
│  │ - 数据格式错误 → 尝试自动转换      │         │
│  │ - 分析失败 → 返回原始数据摘要      │         │
│  └────────────────────────────────────┘         │
│                                                 │
│  依赖Skills：file_handling, code_execution      │
│  标签：#数据分析 #可视化 #报告                   │
│                                                 │
```

## 二、Skill 系统设计

### 2.1 Skill 定义

```python
"""
Agent Skill 基础框架
"""

from dataclasses import dataclass, field
from typing import Callable, Any
from enum import Enum


class SkillStatus(Enum):
    READY = "ready"
    RUNNING = "running"
    SUCCESS = "success"
    FAILED = "failed"


@dataclass
class SkillStep:
    """技能的一个执行步骤"""
    name: str
    tool: str                           # 使用的工具名
    params_template: dict               # 参数模板（可包含变量引用）
    description: str = ""
    retry_count: int = 1                # 失败重试次数


@dataclass
class Skill:
    """可组合的Agent技能"""
    name: str
    description: str
    steps: list[SkillStep]
    preconditions: list[str] = field(default_factory=list)
    postconditions: list[str] = field(default_factory=list)
    fallback_skill: str | None = None   # 失败时回退到的Skill
    tags: list[str] = field(default_factory=list)
    dependencies: list[str] = field(default_factory=list)  # 依赖的其他Skill

    def to_prompt(self) -> str:
        """将Skill描述转换为LLM可理解的文本"""
        steps_text = "\n".join(
            f"  {i+1}. {s.name}（工具: {s.tool}）- {s.description}"
            for i, s in enumerate(self.steps)
        )
        return f"""技能: {self.name}
描述: {self.description}
前置条件: {', '.join(self.preconditions) or '无'}
执行步骤:
{steps_text}
完成标准: {', '.join(self.postconditions) or '无'}"""
```

### 2.2 Skill Registry（技能注册表）

```python
"""
Skill Registry —— 技能的发现与管理
"""

from dataclasses import dataclass, field


@dataclass
class SkillRegistry:
    """技能注册表：管理所有可用技能"""

    _skills: dict[str, Skill] = field(default_factory=dict)
    _tag_index: dict[str, list[str]] = field(default_factory=dict)

    def register(self, skill: Skill):
        """注册一个技能"""
        self._skills[skill.name] = skill

        # 建立标签索引
        for tag in skill.tags:
            if tag not in self._tag_index:
                self._tag_index[tag] = []
            self._tag_index[tag].append(skill.name)

    def get(self, name: str) -> Skill | None:
        """按名称获取技能"""
        return self._skills.get(name)

    def search_by_tag(self, tag: str) -> list[Skill]:
        """按标签搜索技能"""
        names = self._tag_index.get(tag, [])
        return [self._skills[n] for n in names if n in self._skills]

    def search_by_description(self, query: str, llm_client=None) -> list[Skill]:
        """
        语义搜索技能 —— 让LLM根据用户意图匹配最相关的技能
        """
        if llm_client is None:
            # 简单的关键词匹配回退
            return [
                s for s in self._skills.values()
                if query.lower() in s.description.lower()
            ]

        # 用LLM做语义匹配
        skills_desc = "\n".join(
            f"- {s.name}: {s.description}" for s in self._skills.values()
        )
        response = llm_client.chat.completions.create(
            model="gpt-4o-mini",
            messages=[{
                "role": "user",
                "content": f"从以下技能中选出最匹配用户意图的（返回技能名列表，JSON数组）：\n"
                           f"用户意图：{query}\n\n可用技能：\n{skills_desc}"
            }],
            response_format={"type": "json_object"},
        )
        import json
        matched = json.loads(response.choices[0].message.content)
        names = matched.get("skills", [])
        return [self._skills[n] for n in names if n in self._skills]

    def list_all(self) -> list[str]:
        """列出所有技能名"""
        return list(self._skills.keys())

    def get_skill_graph(self) -> dict[str, list[str]]:
        """获取技能依赖图"""
        return {
            name: skill.dependencies
            for name, skill in self._skills.items()
        }
```

### 2.3 Skill Executor（技能执行器）

```python
"""
Skill Executor —— 执行技能的运行时
"""

from dataclasses import dataclass, field


@dataclass
class SkillContext:
    """技能执行上下文 —— 在步骤间传递数据"""
    variables: dict[str, Any] = field(default_factory=dict)
    step_results: list[dict] = field(default_factory=list)
    status: SkillStatus = SkillStatus.READY

    def set(self, key: str, value: Any):
        self.variables[key] = value

    def get(self, key: str, default=None):
        return self.variables.get(key, default)


class SkillExecutor:
    """技能执行器"""

    def __init__(self, registry: SkillRegistry, tool_runner: Callable):
        self.registry = registry
        self.tool_runner = tool_runner  # 实际执行工具的函数

    async def execute(self, skill_name: str, initial_params: dict) -> SkillContext:
        """执行一个技能"""
        skill = self.registry.get(skill_name)
        if not skill:
            raise ValueError(f"技能 '{skill_name}' 未注册")

        ctx = SkillContext()
        ctx.variables.update(initial_params)
        ctx.status = SkillStatus.RUNNING

        # 检查前置条件
        if not self._check_preconditions(skill, ctx):
            ctx.status = SkillStatus.FAILED
            if skill.fallback_skill:
                print(f"前置条件不满足，回退到技能: {skill.fallback_skill}")
                return await self.execute(skill.fallback_skill, initial_params)
            return ctx

        # 逐步执行
        for step in skill.steps:
            result = await self._execute_step(step, ctx)
            ctx.step_results.append({
                "step": step.name,
                "tool": step.tool,
                "result": result,
                "success": result is not None,
            })

            if result is None:
                # 步骤失败
                if skill.fallback_skill:
                    print(f"步骤 '{step.name}' 失败，回退到技能: {skill.fallback_skill}")
                    return await self.execute(skill.fallback_skill, initial_params)
                ctx.status = SkillStatus.FAILED
                return ctx

            # 将结果存入上下文供后续步骤使用
            ctx.set(f"step_{step.name}_result", result)

        # 检查后置条件
        if self._check_postconditions(skill, ctx):
            ctx.status = SkillStatus.SUCCESS
        else:
            ctx.status = SkillStatus.FAILED

        return ctx

    async def _execute_step(self, step: SkillStep, ctx: SkillContext) -> Any:
        """执行单个步骤，支持重试"""
        params = self._resolve_params(step.params_template, ctx)

        for attempt in range(step.retry_count):
            try:
                result = await self.tool_runner(step.tool, params)
                return result
            except Exception as e:
                print(f"步骤 '{step.name}' 第{attempt+1}次执行失败: {e}")
                if attempt == step.retry_count - 1:
                    return None

    def _resolve_params(self, template: dict, ctx: SkillContext) -> dict:
        """解析参数模板中的变量引用"""
        resolved = {}
        for key, value in template.items():
            if isinstance(value, str) and value.startswith("$"):
                # 从上下文获取变量值
                var_name = value[1:]
                resolved[key] = ctx.get(var_name, value)
            else:
                resolved[key] = value
        return resolved

    def _check_preconditions(self, skill: Skill, ctx: SkillContext) -> bool:
        """检查前置条件（简化版本）"""
        # 实际项目中可以用LLM评估条件是否满足
        return True

    def _check_postconditions(self, skill: Skill, ctx: SkillContext) -> bool:
        """检查后置条件（简化版本）"""
        return True
```

## 三、技能组合与编排

### 3.1 技能组合模式

```
三种Skill组合方式：

1. 顺序组合（Sequential）
   ┌───────┐   ┌───────┐   ┌───────┐
   │Skill A│──►│Skill B│──►│Skill C│
   └───────┘   └───────┘   └───────┘
   A的输出是B的输入

2. 并行组合（Parallel）
   ┌───────┐
   │Skill A│──┐
   └───────┘  │   ┌────────┐
              ├──►│ 合并   │
   ┌───────┐  │   └────────┘
   │Skill B│──┘
   └───────┘
   A和B同时执行，结果合并

3. 条件组合（Conditional）
              ┌── 条件满足 ──► Skill A
   判断条件 ──┤
              └── 条件不满足 ─► Skill B
   根据上下文选择不同Skill
```

### 3.2 组合技能实现

```python
"""
技能组合 —— 将多个Skill编排为复合Skill
"""


def create_composite_skill(
    name: str,
    description: str,
    sub_skills: list[str],
    mode: str = "sequential"
) -> Skill:
    """
    创建组合技能

    Args:
        name: 组合技能名
        sub_skills: 子技能名列表
        mode: sequential | parallel | conditional
    """
    steps = []
    for i, skill_name in enumerate(sub_skills):
        steps.append(SkillStep(
            name=f"执行_{skill_name}",
            tool=f"__skill__{skill_name}",  # 特殊工具标识，执行器识别后递归调用
            params_template={"input": f"$step_{i-1}_result" if i > 0 else "$input"},
            description=f"执行子技能 {skill_name}",
        ))

    return Skill(
        name=name,
        description=description,
        steps=steps,
        dependencies=sub_skills,
        tags=["composite"],
    )


# ===== 使用示例 =====

# 基础技能
web_search = Skill(
    name="web_search",
    description="搜索互联网获取信息",
    steps=[SkillStep("搜索", "search_engine", {"query": "$query"})],
    tags=["搜索", "信息获取"],
)

summarize = Skill(
    name="summarize",
    description="对文本进行摘要",
    steps=[SkillStep("摘要", "llm_generate", {"prompt": "摘要以下内容: $text"})],
    tags=["文本处理", "摘要"],
)

write_report = Skill(
    name="write_report",
    description="撰写结构化报告",
    steps=[
        SkillStep("拟提纲", "llm_generate", {"prompt": "为以下内容拟提纲: $content"}),
        SkillStep("写正文", "llm_generate", {"prompt": "按提纲撰写报告: $step_拟提纲_result"}),
    ],
    tags=["写作", "报告"],
)

# 组合技能：研究报告 = 搜索 → 摘要 → 写报告
research_report = create_composite_skill(
    name="research_report",
    description="搜索资料、提炼信息、撰写研究报告",
    sub_skills=["web_search", "summarize", "write_report"],
    mode="sequential",
)

# 注册
registry = SkillRegistry()
for skill in [web_search, summarize, write_report, research_report]:
    registry.register(skill)
```

## 四、LLM驱动的技能选择

### 4.1 让Agent自主选择技能

```python
"""
Agent根据用户意图自动匹配和执行Skill
"""

from openai import OpenAI
import json


class SkillAgent:
    """基于Skill的Agent"""

    def __init__(self, registry: SkillRegistry, executor: SkillExecutor):
        self.registry = registry
        self.executor = executor
        self.client = OpenAI()

    async def handle(self, user_request: str) -> str:
        """处理用户请求：选择技能 → 执行 → 返回结果"""

        # 1. 让LLM选择最合适的技能
        skill = await self._select_skill(user_request)
        if not skill:
            return "抱歉，没有找到匹配的技能来完成这个请求。"

        print(f"选中技能: {skill.name}")

        # 2. 让LLM提取执行参数
        params = await self._extract_params(user_request, skill)

        # 3. 执行技能
        ctx = await self.executor.execute(skill.name, params)

        # 4. 用LLM整理结果
        if ctx.status == SkillStatus.SUCCESS:
            return await self._format_result(user_request, ctx)
        else:
            return f"技能 '{skill.name}' 执行失败，请检查输入或换种方式描述需求。"

    async def _select_skill(self, request: str) -> Skill | None:
        """LLM选择技能"""
        all_skills = [
            self.registry.get(name) for name in self.registry.list_all()
        ]
        skills_prompt = "\n".join(s.to_prompt() for s in all_skills if s)

        response = self.client.chat.completions.create(
            model="gpt-4o-mini",
            response_format={"type": "json_object"},
            messages=[{
                "role": "user",
                "content": f"""从以下技能中选出最适合处理用户请求的一个。

用户请求：{request}

可用技能：
{skills_prompt}

返回JSON：{{"skill_name": "选中的技能名", "reason": "选择原因"}}
如果没有合适的技能，返回：{{"skill_name": null, "reason": "原因"}}"""
            }]
        )
        result = json.loads(response.choices[0].message.content)
        name = result.get("skill_name")
        return self.registry.get(name) if name else None

    async def _extract_params(self, request: str, skill: Skill) -> dict:
        """LLM从请求中提取技能所需参数"""
        response = self.client.chat.completions.create(
            model="gpt-4o-mini",
            response_format={"type": "json_object"},
            messages=[{
                "role": "user",
                "content": f"""从用户请求中提取执行技能所需的参数。

用户请求：{request}
技能：{skill.to_prompt()}

返回JSON对象，包含技能执行需要的所有参数。"""
            }]
        )
        return json.loads(response.choices[0].message.content)

    async def _format_result(self, request: str, ctx: SkillContext) -> str:
        """LLM整理执行结果"""
        results = "\n".join(
            f"步骤 {r['step']}: {r['result']}"
            for r in ctx.step_results
        )
        response = self.client.chat.completions.create(
            model="gpt-4o",
            messages=[{
                "role": "user",
                "content": f"用户请求: {request}\n\n执行结果:\n{results}\n\n"
                           f"请整理成用户友好的回答。"
            }]
        )
        return response.choices[0].message.content
```

### 5.2 Semantic Kernel 的 Skill 模型

```python
"""
Microsoft Semantic Kernel 中 Skill 的使用方式
—— 业界最早将 Skill 作为一等公民的Agent框架
"""

import semantic_kernel as sk
from semantic_kernel.functions import kernel_function


# 定义一个 Skill（Plugin）
class WritingSkill:
    """写作技能插件"""

    @kernel_function(
        name="summarize",
        description="对文本进行摘要"
    )
    def summarize(self, text: str) -> str:
        # 使用LLM进行摘要
        return f"摘要：{text[:100]}..."

    @kernel_function(
        name="translate",
        description="翻译文本"
    )
    def translate(self, text: str, target_language: str = "English") -> str:
        return f"翻译({target_language})：{text}"

    @kernel_function(
        name="rewrite_formal",
        description="将文本改写为正式风格"
    )
    def rewrite_formal(self, text: str) -> str:
        return f"正式版本：{text}"


# 注册 Skill
kernel = sk.Kernel()
kernel.add_plugin(WritingSkill(), plugin_name="writing")

# Agent 可以自动发现并使用这些 Skill
# kernel.invoke("writing", "summarize", text="...")
```

## 六、Skill vs Tool vs Agent

```
三者关系辨析：

┌─────────────────────────────────────────────────────┐
│                                                     │
│  Tool（工具）                                       │
│    → 最小执行单元                                    │
│    → 一个函数：输入 → 输出                           │
│    → 例：search(query) → results                    │
│                                                     │
│  Skill（技能）                                      │
│    → 由一个或多个Tool + 执行逻辑组成                 │
│    → 包含：前置条件、执行步骤、后置验证、回退策略    │
│    → 例：research_and_report = search → summarize    │
│          → write_report                              │
│                                                     │
│  Agent（智能体）                                    │
│    → 拥有一组Skills的自主决策实体                    │
│    → 根据目标自主选择和组合Skills                    │
│    → 例：分析师Agent 拥有 data_analysis,             │
│          research_report, visualization 等Skills     │
│                                                     │
│  层级关系：                                          │
│    Tools ──组成──► Skills ──装备──► Agent             │
│                                                     │
└─────────────────────────────────────────────────────┘
```

## 七、小结

```
Agent Skills 要点回顾：

┌───────────────────────────────────────────────┐
│  1. Skill 是什么                               │
│     → 比Tool更高层的能力抽象                    │
│     → 包含执行逻辑、条件检查、错误处理         │
│                                               │
│  2. 核心组成                                   │
│     → 前置条件 + 执行步骤 + 后置验证 + 回退    │
│     → Skill Registry 管理发现与检索            │
│     → Skill Executor 负责执行与编排            │
│                                               │
│  3. 组合能力                                   │
```
