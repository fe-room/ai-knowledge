# 3.5.1 Model Context Protocol（MCP）

> 理解AI工具集成的标准化协议

## 一、为什么需要MCP？

### 1.1 传统工具集成的痛点

```
传统方式 —— 每个Agent × 每个工具 = N×M 种集成：

┌──────────┐     ┌────────────┐
│ LangChain│────→│ 天气API     │  自定义适配器
│          │────→│ 数据库查询   │  自定义适配器
│          │────→│ 文件读写     │  自定义适配器
└──────────┘     └────────────┘

┌──────────┐     ┌────────────┐
│  CrewAI  │────→│ 天气API     │  又写一遍适配器
│          │────→│ 数据库查询   │  又写一遍适配器
└──────────┘     └────────────┘

问题：
  ❌ 同一个工具要为不同框架写多次适配
  ❌ 工具发现、权限控制各自实现
  ❌ 缺乏标准的输入/输出格式
  ❌ 工具更新时所有集成都要改
```

### 1.2 MCP的解决思路

```
MCP方式 —— 标准协议，即插即用：

┌──────────────┐         ┌──────────────┐
│  MCP Client  │ ◄─MCP─► │  MCP Server  │
│ (Agent/IDE)  │  协议    │  (天气工具)   │
└──────────────┘         └──────────────┘
                          ┌──────────────┐
        同一协议 ◄─MCP─►  │  MCP Server  │
                          │  (数据库工具) │
                          └──────────────┘

类比：
  USB之前：每个设备需要专用接口
  USB之后：统一接口，即插即用

  MCP之前：每个工具需要专用集成代码
  MCP之后：标准协议，任何Client可调用任何Server
```

### 1.3 MCP核心特征

| 特征 | 说明 |
|------|------|
| 开放标准 | Anthropic提出，开源协议，不绑定特定厂商 |
| 双向通信 | 基于JSON-RPC 2.0，Client和Server可双向交互 |
| 三大原语 | Tools（工具）、Resources（资源）、Prompts（提示词模板） |
| 传输灵活 | 支持stdio（本地进程）和SSE（远程HTTP）两种传输方式 |
| 安全模型 | 内置权限协商和用户确认机制 |

## 二、MCP架构详解

### 2.1 整体架构

```
┌─────────────────────────────────────────────────────────┐
│                    Host 应用                             │
│  （Claude Desktop / IDE / 自定义Agent）                   │
│                                                         │
│   ┌─────────────┐  ┌─────────────┐  ┌─────────────┐    │
│   │ MCP Client1 │  │ MCP Client2 │  │ MCP Client3 │    │
│   └──────┬──────┘  └──────┬──────┘  └──────┬──────┘    │
│          │                │                │            │
└──────────┼────────────────┼────────────────┼────────────┘
           │ JSON-RPC       │ JSON-RPC       │ JSON-RPC
           │                │                │
    ┌──────↓──────┐  ┌──────↓──────┐  ┌──────↓──────┐
    │ MCP Server  │  │ MCP Server  │  │ MCP Server  │
    │ （文件系统） │  │ （数据库）   │  │ （Web搜索）  │
    └─────────────┘  └─────────────┘  └─────────────┘
```

**三层架构**：
- **Host**：宿主应用，管理多个Client实例
- **Client**：与单个Server保持一对一连接，处理协议通信
- **Server**：暴露工具/资源/提示词，由独立进程或远程服务提供

### 2.2 三大原语

```python
"""
MCP 三大能力原语
"""

# 1. Tools（工具）—— 模型可调用的函数
#    由Server定义，由Client发现，由LLM决定何时调用
{
    "name": "query_database",
    "description": "查询数据库中的用户信息",
    "inputSchema": {
        "type": "object",
        "properties": {
            "user_id": {"type": "string", "description": "用户ID"}
        },
        "required": ["user_id"]
    }
}

# 2. Resources（资源）—— 可读取的数据源
#    类似文件系统：提供URI寻址的只读数据
{
    "uri": "db://users/profile",
    "name": "用户档案",
    "mimeType": "application/json"
}

# 3. Prompts（提示词模板）—— 可复用的交互模板
#    Server预定义好的提示词方案
{
    "name": "code_review",
    "description": "代码审查提示词",
    "arguments": [
        {"name": "language", "required": True},
        {"name": "code", "required": True}
    ]
}
```

### 2.3 通信流程

```
Client                              Server
  │                                   │
  │ ──── initialize ──────────────►   │   1. 协议握手
  │ ◄─── capabilities ────────────   │      交换能力信息
  │                                   │
  │ ──── tools/list ──────────────►   │   2. 发现工具
  │ ◄─── [tool1, tool2, ...] ─────   │      获取可用工具列表
  │                                   │
  │ ──── tools/call ──────────────►   │   3. 调用工具
  │      {name, arguments}            │      LLM决策后执行
  │ ◄─── {content: result} ───────   │
  │                                   │
  │ ──── resources/read ──────────►   │   4. 读取资源
  │      {uri}                        │      按需获取数据
  │ ◄─── {contents: [...]} ───────   │
  │                                   │
```

## 三、MCP Server 开发

### 3.1 Python MCP Server

```python
"""
MCP Server 示例：知识库查询工具
使用官方 mcp 库
"""

from mcp.server import Server
from mcp.server.stdio import stdio_server
from mcp.types import (
    Tool,
    TextContent,
    Resource,
)
import json

# 创建Server实例
server = Server("knowledge-base")

# ===== 定义工具 =====

@server.tool()
async def search_docs(query: str, top_k: int = 3) -> str:
    """
    在知识库中搜索相关文档。

    Args:
        query: 搜索查询词
        top_k: 返回结果数量
    """
    # 实际项目中这里连接向量数据库
    # 示例使用模拟数据
    results = [
        {"title": "Agent架构设计", "score": 0.95, "snippet": "Agent由感知、决策、执行三部分组成..."},
        {"title": "RAG最佳实践", "score": 0.87, "snippet": "检索增强生成需要关注分块策略..."},
        {"title": "Prompt工程指南", "score": 0.82, "snippet": "好的提示词应该清晰、具体..."},
    ]
    return json.dumps(results[:top_k], ensure_ascii=False, indent=2)


@server.tool()
async def add_document(title: str, content: str, tags: list[str] | None = None) -> str:
    """
    向知识库添加新文档。

    Args:
        title: 文档标题
        content: 文档内容
        tags: 标签列表（可选）
    """
    # 参数校验
    if not title.strip() or not content.strip():
        return json.dumps({"error": "标题和内容不能为空"})

    doc_id = f"doc_{hash(title) % 10000:04d}"

    return json.dumps({
        "status": "success",
        "doc_id": doc_id,
        "message": f"文档 '{title}' 已添加"
    }, ensure_ascii=False)


# ===== 定义资源 =====

@server.resource("kb://stats")
async def get_stats() -> str:
    """知识库统计信息"""
    return json.dumps({
        "total_docs": 1523,
        "total_chunks": 8934,
        "last_updated": "2025-04-01"
    })


# ===== 启动Server =====

async def main():
    async with stdio_server() as (read, write):
        await server.run(read, write)

if __name__ == "__main__":
    import asyncio
    asyncio.run(main())
```

### 3.2 JavaScript/TypeScript MCP Server

```typescript
/**
 * MCP Server 示例：天气查询工具（TypeScript）
 */

import { Server } from "@modelcontextprotocol/sdk/server/index.js";
import { StdioServerTransport } from "@modelcontextprotocol/sdk/server/stdio.js";
import {
  CallToolRequestSchema,
  ListToolsRequestSchema,
} from "@modelcontextprotocol/sdk/types.js";

const server = new Server(
  { name: "weather-server", version: "1.0.0" },
  { capabilities: { tools: {} } }
);

// 列出可用工具
server.setRequestHandler(ListToolsRequestSchema, async () => ({
  tools: [
    {
      name: "get_weather",
      description: "获取指定城市的天气信息",
      inputSchema: {
        type: "object" as const,
        properties: {
          city: { type: "string", description: "城市名称" },
          unit: {
            type: "string",
            enum: ["celsius", "fahrenheit"],
            description: "温度单位",
          },
        },
        required: ["city"],
      },
    },
  ],
}));

// 处理工具调用
server.setRequestHandler(CallToolRequestSchema, async (request) => {
  const { name, arguments: args } = request.params;

  if (name === "get_weather") {
    const city = args?.city as string;
    // 实际项目中调用天气API
    return {
      content: [
        {
          type: "text" as const,
          text: JSON.stringify({
            city,
            temperature: 25,
            condition: "晴",
            humidity: 60,
          }),
        },
      ],
    };
  }

  throw new Error(`未知工具: ${name}`);
});

// 启动
async function main() {
  const transport = new StdioServerTransport();
  await server.connect(transport);
}

main().catch(console.error);
```

## 四、MCP Client 开发

### 4.1 Python Client —— 连接MCP Server的Agent

```python
"""
在自定义Agent中作为MCP Client调用MCP Server
"""

from mcp import ClientSession
from mcp.client.stdio import stdio_client, StdioServerParameters
from openai import OpenAI
import json

class MCPAgent:
    """通过MCP协议调用工具的Agent"""

    def __init__(self):
        self.llm = OpenAI()
        self.sessions: dict[str, ClientSession] = {}

    async def connect_server(self, name: str, command: str, args: list[str]):
        """连接一个MCP Server"""
        params = StdioServerParameters(command=command, args=args)
        read, write = await stdio_client(params).__aenter__()
        session = ClientSession(read, write)
        await session.initialize()
        self.sessions[name] = session
        print(f"已连接 Server: {name}")

        # 列出可用工具
        tools = await session.list_tools()
        for tool in tools.tools:
            print(f"  工具: {tool.name} - {tool.description}")

    async def get_all_tools(self) -> list[dict]:
        """收集所有Server的工具，转换为OpenAI Function格式"""
        functions = []
        for server_name, session in self.sessions.items():
            tools = await session.list_tools()
            for tool in tools.tools:
                functions.append({
                    "type": "function",
                    "function": {
                        "name": f"{server_name}__{tool.name}",
                        "description": tool.description,
                        "parameters": tool.inputSchema,
                    }
                })
        return functions

    async def call_tool(self, full_name: str, arguments: dict) -> str:
        """调用MCP工具"""
        server_name, tool_name = full_name.split("__", 1)
        session = self.sessions.get(server_name)
        if not session:
            return f"错误：未找到Server '{server_name}'"

        result = await session.call_tool(tool_name, arguments)
        # 提取文本内容
        return "".join(
            c.text for c in result.content if hasattr(c, "text")
        )

    async def chat(self, user_message: str) -> str:
        """完整的对话循环"""
        tools = await self.get_all_tools()
        messages = [{"role": "user", "content": user_message}]

        while True:
            response = self.llm.chat.completions.create(
                model="gpt-4o",
                messages=messages,
                tools=tools if tools else None,
            )
            choice = response.choices[0]

            # 如果没有工具调用，返回最终回答
            if choice.finish_reason != "tool_calls":
                return choice.message.content

            # 处理工具调用
            messages.append(choice.message)
            for tool_call in choice.message.tool_calls:
                result = await self.call_tool(
                    tool_call.function.name,
                    json.loads(tool_call.function.arguments)
                )
                messages.append({
                    "role": "tool",
                    "tool_call_id": tool_call.id,
                    "content": result
                })


# ===== 使用示例 =====
async def main():
    agent = MCPAgent()

    # 连接多个MCP Server
    await agent.connect_server(
        "kb", "python", ["knowledge_server.py"]
    )
    await agent.connect_server(
        "weather", "node", ["weather_server.js"]
    )

    # Agent自动发现所有工具并按需调用
    answer = await agent.chat("帮我查一下北京天气，然后在知识库里搜索天气相关的文档")
    print(answer)
```

### 4.2 MCP与传统Function Calling对比

```
传统 Function Calling：
┌──────────────────────────────────────────────┐
│  开发者手动定义每个工具的Schema              │
│  开发者手动实现每个工具的调用逻辑            │
│  工具列表硬编码在代码中                       │
│  换一个Agent框架 → 重新集成所有工具          │
└──────────────────────────────────────────────┘

MCP方式：
┌──────────────────────────────────────────────┐
│  Server自描述工具Schema → Client自动发现     │
│  工具实现封装在Server中 → Client只管调用     │
│  工具列表动态获取 → 插拔任意Server           │
│  换Agent框架 → 只需实现MCP Client协议       │
└──────────────────────────────────────────────┘
```

## 五、MCP与Agent框架集成

### 5.1 LangChain + MCP

```python
"""
将MCP工具注入LangChain Agent
"""

from langchain_core.tools import StructuredTool
from langchain_openai import ChatOpenAI
from langchain.agents import AgentExecutor, create_openai_tools_agent
from langchain_core.prompts import ChatPromptTemplate

async def create_langchain_agent_with_mcp(session: ClientSession):
    """从MCP Server动态创建LangChain工具"""

    # 1. 从MCP获取工具列表
    mcp_tools = await session.list_tools()

    # 2. 转换为LangChain工具
    lc_tools = []
    for tool in mcp_tools.tools:
        async def _call(arguments, _tool_name=tool.name, **kwargs):
            result = await session.call_tool(_tool_name, arguments)
            return "".join(c.text for c in result.content if hasattr(c, "text"))

        lc_tools.append(StructuredTool(
            name=tool.name,
            description=tool.description,
            func=_call,
            args_schema=None,  # 实际项目中从inputSchema生成Pydantic模型
        ))

    # 3. 构建LangChain Agent
    llm = ChatOpenAI(model="gpt-4o")
    prompt = ChatPromptTemplate.from_messages([
        ("system", "你是一个智能助手，可以使用工具完成用户请求。"),
        ("human", "{input}"),
        ("placeholder", "{agent_scratchpad}"),
    ])
    agent = create_openai_tools_agent(llm, lc_tools, prompt)
    return AgentExecutor(agent=agent, tools=lc_tools)
```

async def connect_remote_server():
    """连接远程MCP Server"""

## 六、MCP安全与最佳实践

### 6.1 安全注意事项

```
⚠️ MCP安全原则：

1. 最小权限
   - Server只暴露必要的工具和资源
   - 不要在Server中放置敏感操作（如删库、转账）而不做二次确认

2. 输入校验
   - Server端必须校验所有输入参数
   - 不信任Client传来的任何数据（LLM的工具调用参数可能不合理）

3. 用户确认
   - 高危操作应返回确认提示，而非直接执行
   - Host应用负责向用户展示确认对话框

4. 传输安全
   - stdio模式：Server作为子进程，天然隔离
   - SSE模式：必须使用HTTPS + 认证Token
```

### 6.2 Server开发最佳实践

生产级MCP Server应包含的要素

    Args:

    if not doc_id or not doc_id.startswith("doc_"):

    # ✅ 3. 返回确认信息（而非直接删除）

        "action": "confirm_delete",
        "doc_id": doc_id,
        "message": "确认要删除此文档？此操作不可撤销。",

## 七、小结

```
MCP 要点回顾：

┌───────────────────────────────────────────────┐
│  1. MCP是什么                                  │
│     → 工具集成的开放标准协议                    │
│     → Client-Server架构 + JSON-RPC通信         │
│                                               │
│  2. 三大原语                                   │
│     → Tools（工具函数）                        │
│     → Resources（数据资源）                    │
│     → Prompts（提示词模板）                    │
│                                               │
│  3. 核心价值                                   │
│     → 写一次Server，所有Client可用             │
│     → 工具发现自动化                           │
│     → 与框架解耦                               │
│                                               │
│  4. 适用场景                                   │
│     → 需要跨多个Agent/应用共享工具             │
│     → IDE集成（Cursor、VS Code Copilot）       │
│     → 企业内部工具标准化                       │
│                                               │
│  5. 不适用场景                                 │
│     → 简单的单Agent单工具场景（过度设计）       │
│     → 需要超低延迟的工具调用（协议有开销）      │
└───────────────────────────────────────────────┘
```
