# 3.4.3 云平台部署

> 将Agent应用部署到云平台

## 一、云平台选择

### 1.1 主流云平台

```
云平台对比：

| 平台 | 特点 | 适用场景 |
|------|------|---------|
| AWS | 功能最全，成熟度高 | 企业级应用 |
| Azure | 与微软生态集成 | 企业应用 |
| GCP | AI/ML支持强 | AI应用 |
| 阿里云 | 国内访问快 | 国内应用 |
| 腾讯云 | 价格优惠 | 中小应用 |

服务类型：
- IaaS：虚拟机（EC2, ECS）
- PaaS：应用服务（Elastic Beanstalk, App Service）
- FaaS：无服务器（Lambda, Cloud Functions）
```

### 1.2 部署方式选择

```
部署方式：

1. 虚拟机部署
   优点：完全控制，灵活
   缺点：运维成本高

2. 容器服务
   优点：易于扩展，标准化
   缺点：需要学习容器编排

3. 无服务器
   优点：无需管理服务器，按需付费
   缺点：冷启动，限制多

4. 应用平台
   优点：简单易用
   缺点：灵活性较低
```

## 三、云函数部署

### 3.1 Google Cloud Functions

```python
"""
Google Cloud Functions
"""

import functions_framework
from flask import Request, Response

@functions_framework.http
def agent_handler(request: Request):
    """处理HTTP请求"""
    if request.method != 'POST':
        return Response('Method not allowed', status=405)

    data = request.get_json()
    message = data.get('message')

    # Agent处理
    response = agent.chat(message)

    return Response(
        json.dumps({'response': response}),
        status=200,
        mimetype='application/json'
    )
```

```yaml
# 部署命令
gcloud functions deploy agent-api \
    --runtime python311 \
    --trigger-http \
    --entry-point agent_handler \
    --set-env-vars OPENAI_API_KEY=$OPENAI_API_KEY
```

### 3.2 Azure Functions

```python
"""
Azure Functions
"""

import azure.functions as func
import json

def main(req: func.HttpRequest) -> func.HttpResponse:
    """处理HTTP请求"""
    try:
        data = req.get_json()
        message = data.get('message')

        # Agent处理
        response = agent.chat(message)

        return func.HttpResponse(
            json.dumps({'response': response}),
            status_code=200,
            mimetype='application/json'
        )

    except Exception as e:
        return func.HttpResponse(
            json.dumps({'error': str(e)}),
            status_code=500
        )
```

## 四、容器服务部署

### 4.1 AWS ECS + Fargate

```bash
#!/bin/bash
# deploy-to-ecs.sh

# 1. 创建ECR仓库
aws ecr create-repository --repository-name agent-api

# 2. 登录ECR
aws ecr get-login-password --region us-east-1 | \
    docker login --username AWS --password-stdin \
    ACCOUNT.dkr.ecr.us-east-1.amazonaws.com

# 3. 构建并推送镜像
docker build -t agent-api .
docker tag agent-api:latest ACCOUNT.dkr.ecr.us-east-1.amazonaws.com/agent-api:latest
docker push ACCOUNT.dkr.ecr.us-east-1.amazonaws.com/agent-api:latest

# 4. 创建ECS集群
aws ecs create-cluster --cluster-name agent-cluster

# 5. 注册任务定义
aws ecs register-task-definition \
    --cli-input-json file://ecs-task-definition.json

# 6. 创建服务
aws ecs create-service \
    --cluster agent-cluster \
    --service-name agent-api-service \
    --task-definition agent-api \
    --desired-count 2 \
    --launch-type FARGATE \
    --network-configuration "awsvpcConfiguration={subnets=[subnet-xxx],securityGroups=[sg-xxx],assignPublicIp=ENABLED}"

# 7. 创建负载均衡器
aws elbv2 create-load-balancer \
    --name agent-alb \
    --subnets subnet-xxx subnet-yyy
```

### 4.2 Google Cloud Run

```bash
#!/bin/bash
# deploy-to-cloudrun.sh

# 1. 构建镜像
gcloud builds submit --tag gcr.io/PROJECT_ID/agent-api

# 2. 部署到Cloud Run
gcloud run deploy agent-api \
    --image gcr.io/PROJECT_ID/agent-api \
    --platform managed \
    --region us-central1 \
    --allow-unauthenticated \
    --set-env-vars OPENAI_API_KEY=$OPENAI_API_KEY \
    --memory 512Mi \
    --cpu 1

# 3. 获取URL
gcloud run services describe agent-api \
    --platform managed \
    --region us-central1 \
    --format 'value(status.url)'
```

### 4.3 Azure Container Apps

```bash
#!/bin/bash
# deploy-to-containerapps.sh

# 1. 创建资源组
az group create --name agent-rg --location eastus

# 2. 创建容器应用环境
az containerapp env create \
    --name agent-env \
    --resource-group agent-rg \
    --location eastus

# 3. 部署容器应用
az containerapp create \
    --name agent-api \
    --resource-group agent-rg \
    --environment agent-env \
    --image ACCOUNT.azurecr.io/agent-api:latest \
    --target-port 8000 \
    --ingress external \
    --env-vars OPENAI_API_KEY=$OPENAI_API_KEY

# 4. 获取URL
az containerapp show \
    --name agent-api \
    --resource-group agent-rg \
    --query properties.configuration.ingress.fqdn
```

## 五、自动扩缩容

### 5.1 Kubernetes HPA

```yaml
# kubernetes-hpa.yaml

apiVersion: autoscaling/v2
kind: HorizontalPodAutoscaler
metadata:
  name: agent-api-hpa
spec:
  scaleTargetRef:
    apiVersion: apps/v1
    kind: Deployment
    name: agent-api
  minReplicas: 2
  maxReplicas: 10
  metrics:
    - type: Resource
      resource:
        name: cpu
        target:
          type: Utilization
          averageUtilization: 70
    - type: Resource
      resource:
        name: memory
        target:
          type: Utilization
          averageUtilization: 80
  behavior:
    scaleDown:
      stabilizationWindowSeconds: 300
      policies:
        - type: Percent
          value: 10
          periodSeconds: 60
    scaleUp:
      stabilizationWindowSeconds: 0
      policies:
        - type: Percent
          value: 100
          periodSeconds: 15
```

### 5.2 AWS Auto Scaling

```bash
# 配置应用自动扩缩容

# 创建目标组
aws application-autoscaling register-scalable-target \
    --service-namespace ecs \
    --resource-id service/agent-cluster/agent-api-service \
    --scalable-dimension ecs:service:DesiredCount \
    --min-capacity 2 \
    --max-capacity 10

# 创建扩缩容策略
aws application-autoscaling put-scaling-policy \
    --service-namespace ecs \
    --resource-id service/agent-cluster/agent-api-service \
    --scalable-dimension ecs:service:DesiredCount \
    --policy-name agent-api-scaling-policy \
    --policy-type TargetTrackingScaling \
    --target-tracking-scaling-policy-configuration file://scaling-policy.json

# scaling-policy.json
{
    "TargetValue": 70.0,
    "PredefinedMetricSpecification": {
        "PredefinedMetricType": "ECSServiceAverageCPUUtilization"
    },
    "ScaleOutCooldown": 60,
    "ScaleInCooldown": 300
}
```

## 六、域名与SSL

### 6.1 配置自定义域名

```bash
# AWS Route53配置

# 1. 创建托管区域
aws route53 create-hosted-zone \
    --name api.example.com \
    --caller-reference "$(date +%s)"

# 2. 创建A记录指向ALB
aws route53 change-resource-record-sets \
    --hosted-zone-id Z1234567890ABC \
    --change-batch file://dns-record.json

# dns-record.json
{
    "Changes": [{
        "Action": "CREATE",
        "ResourceRecordSet": {
            "Name": "api.example.com",
            "Type": "A",
            "AliasTarget": {
                "HostedZoneId": "Z215JYRZR1TBD5",
                "DNSName": "agent-alb-xxx.us-east-1.elb.amazonaws.com",
                "EvaluateTargetHealth": true
            }
        }
    }]
}
```

## 七、监控与日志

### 7.1 CloudWatch监控

```python
"""
CloudWatch监控
"""

import boto3
from datetime import datetime

cloudwatch = boto3.client('cloudwatch')

def put_metric(metric_name, value, unit='Count'):
    """发送指标"""
    cloudwatch.put_metric_data(
        Namespace='AgentAPI',
        MetricData=[{
            'MetricName': metric_name,
            'Value': value,
            'Unit': unit,
            'Timestamp': datetime.now()
        }]
    )

# 使用
put_metric('RequestCount', 1)
put_metric('Latency', 150, 'Milliseconds')
put_metric('ErrorCount', 0)
```

### 7.2 日志收集

```python
"""
CloudWatch日志
"""

import logging
from watchtower import CloudWatchLogHandler

# 配置日志
logger = logging.getLogger(__name__)
logger.setLevel(logging.INFO)

# 添加CloudWatch处理器
handler = CloudWatchLogHandler(
    log_group='/aws/ecs/agent-api',
    stream_name='app-logs'
)
logger.addHandler(handler)

# 使用
logger.info("Request received", extra={
    'request_id': 'xxx',
    'path': '/chat'
})
```

### 核心要点

1. **云平台选择**：根据需求选择合适平台
2. **部署方式**：VM、容器服务、无服务器

- 使用托管服务减少运维
- 配置自动扩缩容
- 实现完整的监控告警
- 做好备份和灾难恢复
