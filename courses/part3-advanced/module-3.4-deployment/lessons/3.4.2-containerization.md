# 3.4.2 容器化部署

> 使用Docker部署Agent应用

## 一、Docker基础

### 1.1 为什么容器化？

```
容器化优势：

1. 环境一致性
   - 开发、测试、生产环境相同
   - 避免"在我机器上能运行"

2. 部署便捷
   - 一键部署
   - 快速扩缩容

3. 资源隔离
   - 独立的运行环境
   - 资源限制

4. 版本管理
   - 镜像版本化
   - 回滚容易
```

### 1.2 Docker核心概念

```
Docker核心概念：

镜像（Image）
  └── 只读模板，包含应用和依赖

容器（Container）
  └── 镜像的运行实例

仓库（Registry）
  └── 存储和分发镜像

Dockerfile
  └── 构建镜像的脚本
```

## 二、Dockerfile编写

### 2.1 Python应用Dockerfile

```dockerfile
# Python Agent应用Dockerfile

# 基础镜像
FROM python:3.11-slim

# 设置工作目录
WORKDIR /app

# 设置环境变量
ENV PYTHONDONTWRITEBYTECODE=1 \
    PYTHONUNBUFFERED=1 \
    PIP_NO_CACHE_DIR=1

# 安装系统依赖
RUN apt-get update && apt-get install -y --no-install-recommends \
    build-essential \
    && rm -rf /var/lib/apt/lists/*

# 复制依赖文件
COPY requirements.txt .

# 安装Python依赖
RUN pip install --no-cache-dir -r requirements.txt

# 复制应用代码
COPY . .

# 创建非root用户
RUN useradd -m -u 1000 appuser && \
    chown -R appuser:appuser /app
USER appuser

# 暴露端口
EXPOSE 8000

# 健康检查
HEALTHCHECK --interval=30s --timeout=10s --start-period=5s --retries=3 \
    CMD curl -f http://localhost:8000/health || exit 1

# 启动命令
CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000"]
```

### 2.2 Node.js应用Dockerfile

```dockerfile
# Node.js Agent应用Dockerfile

# 构建阶段
FROM node:18-alpine AS builder

WORKDIR /app

# 复制依赖文件
COPY package*.json ./

# 安装依赖
RUN npm ci --only=production

# 复制源代码
COPY . .

# 运行阶段
FROM node:18-alpine

WORKDIR /app

# 复制构建产物
COPY --from=builder /app/node_modules ./node_modules
COPY --from=builder /app/package*.json ./
COPY --from=builder /app/dist ./dist
COPY --from=builder /app/src ./src

# 创建非root用户
RUN addgroup -g 1001 -S nodejs && \
    adduser -S nodejs -u 1001
USER nodejs

# 暴露端口
EXPOSE 3000

# 健康检查
HEALTHCHECK --interval=30s --timeout=10s --start-period=5s --retries=3 \
    CMD wget --no-verbose --tries=1 --spider http://localhost:3000/health || exit 1

# 启动命令
CMD ["node", "dist/index.js"]
```

### 2.3 多阶段构建优化

```dockerfile
# 优化的多阶段Dockerfile

# 阶段1: 依赖安装
FROM python:3.11-slim AS deps

WORKDIR /app

COPY requirements.txt .
RUN pip install --user --no-cache-dir -r requirements.txt

# 阶段2: 构建
FROM python:3.11-slim AS builder

WORKDIR /app

# 复制依赖
COPY --from=deps /root/.local /root/.local
ENV PATH=/root/.local/bin:$PATH

# 复制源代码
COPY . .

# 运行测试
RUN python -m pytest tests/ -v

# 阶段3: 生产镜像
FROM python:3.11-slim

WORKDIR /app

# 复制依赖
COPY --from=deps /root/.local /root/.local
ENV PATH=/root/.local/bin:$PATH

# 复制应用
COPY --from=builder /app/app ./app
COPY --from=builder /app/main.py .

# 安全设置
RUN useradd -m -u 1000 appuser && \
    chown -R appuser:appuser /app
USER appuser

EXPOSE 8000

CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000"]
```

## 三、Docker Compose

### 3.1 基础配置

```yaml
# docker-compose.yml

version: '3.8'

services:
  # API服务
  api:
    build:
      context: .
      dockerfile: Dockerfile
    ports:
      - "8000:8000"
    environment:
      - OPENAI_API_KEY=${OPENAI_API_KEY}
      - DATABASE_URL=postgresql://user:pass@db:5432/agent
      - REDIS_URL=redis://redis:6379
    depends_on:
      - db
      - redis
    volumes:
      - ./logs:/app/logs
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:8000/health"]
      interval: 30s
      timeout: 10s
      retries: 3

  # 数据库
  db:
    image: postgres:15-alpine
    environment:
      - POSTGRES_USER=user
      - POSTGRES_PASSWORD=pass
      - POSTGRES_DB=agent
    volumes:
      - postgres_data:/var/lib/postgresql/data
    restart: unless-stopped

  # 缓存
  redis:
    image: redis:7-alpine
    volumes:
      - redis_data:/data
    restart: unless-stopped

  # 向量数据库
  qdrant:
    image: qdrant/qdrant:latest
    ports:
      - "6333:6333"
    volumes:
      - qdrant_data:/qdrant/storage
    restart: unless-stopped

volumes:
  postgres_data:
  redis_data:
  qdrant_data:
```

### 3.2 完整生产配置

```yaml
# docker-compose.prod.yml

version: '3.8'

services:
  # API服务
  api:
    build:
      context: .
      dockerfile: Dockerfile
      args:
        - BUILD_ENV=production
    image: agent-api:latest
    deploy:
      replicas: 3
      resources:
        limits:
          cpus: '1'
          memory: 1G
        reservations:
          cpus: '0.5'
          memory: 512M
      restart_policy:
        condition: on-failure
        delay: 5s
        max_attempts: 3
    environment:
      - APP_ENV=production
      - OPENAI_API_KEY=${OPENAI_API_KEY}
      - DATABASE_URL=postgresql://user:pass@db:5432/agent
      - REDIS_URL=redis://redis:6379
    depends_on:
      db:
        condition: service_healthy
      redis:
        condition: service_started
    networks:
      - backend
    logging:
      driver: "json-file"
      options:
        max-size: "10m"
        max-file: "5"

  # Nginx反向代理
  nginx:
    image: nginx:alpine
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./nginx.conf:/etc/nginx/nginx.conf:ro
      - ./ssl:/etc/nginx/ssl:ro
    depends_on:
      - api
    networks:
      - frontend
      - backend
    restart: always

  # 数据库
  db:
    image: postgres:15-alpine
    environment:
      - POSTGRES_USER=user
      - POSTGRES_PASSWORD=${DB_PASSWORD}
      - POSTGRES_DB=agent
    volumes:
      - postgres_data:/var/lib/postgresql/data
      - ./init.sql:/docker-entrypoint-initdb.d/init.sql
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U user -d agent"]
      interval: 10s
      timeout: 5s
      retries: 5
    networks:
      - backend
    restart: always

  # Redis缓存
  redis:
    image: redis:7-alpine
    command: redis-server --appendonly yes
    volumes:
      - redis_data:/data
    networks:
      - backend
    restart: always

  # 监控
  prometheus:
    image: prom/prometheus:latest
    volumes:
      - ./prometheus.yml:/etc/prometheus/prometheus.yml
      - prometheus_data:/prometheus
    ports:
      - "9090:9090"
    networks:
      - backend

  grafana:
    image: grafana/grafana:latest
    volumes:
      - grafana_data:/var/lib/grafana
    ports:
      - "3001:3000"
    networks:
      - backend

networks:
  frontend:
  backend:

volumes:
  postgres_data:
  redis_data:
  prometheus_data:
  grafana_data:
```

## 四、镜像优化

### 4.1 镜像大小优化

```dockerfile
# 优化后的Dockerfile

# 使用更小的基础镜像
FROM python:3.11-alpine

# 安装必要依赖
RUN apk add --no-cache \
    libffi-dev \
    && rm -rf /var/cache/apk/*

WORKDIR /app

# 先安装依赖（利用缓存）
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# 只复制必要文件
COPY app/ ./app/
COPY main.py .

# 清理
RUN find /usr/local -type d -name __pycache__ -exec rm -rf {} + 2>/dev/null || true

# 使用更小的镜像
FROM python:3.11-alpine

COPY --from=0 /usr/local/lib/python3.11/site-packages /usr/local/lib/python3.11/site-packages
COPY --from=0 /app /app

WORKDIR /app

EXPOSE 8000

CMD ["python", "main.py"]
```

## 五、容器管理

### 5.1 常用命令

```bash
# 构建镜像
docker build -t agent-api:latest .

# 运行容器
docker run -d \
    --name agent-api \
    -p 8000:8000 \
    -e OPENAI_API_KEY=$OPENAI_API_KEY \
    agent-api:latest

# 查看日志
docker logs -f agent-api

# 进入容器
docker exec -it agent-api /bin/sh

# 停止容器
docker stop agent-api

# 删除容器
docker rm agent-api

# 使用Docker Compose
docker-compose up -d
docker-compose logs -f
docker-compose down
docker-compose ps

# 查看资源使用
docker stats
```

### 5.2 健康检查脚本

```bash
#!/bin/bash
# healthcheck.sh

set -e

# 检查API
curl -f http://localhost:8000/health || exit 1

# 检查数据库连接
python -c "from app.db import check_connection; check_connection()" || exit 1

# 检查Redis
python -c "from app.cache import check_redis; check_redis()" || exit 1

exit 0
```

## 六、CI/CD集成

### 6.1 GitHub Actions

```yaml
# .github/workflows/docker-build.yml

name: Build and Push Docker Image

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v3

      - name: Set up Docker Buildx
        uses: docker/setup-buildx-action@v2

      - name: Login to Docker Hub
        uses: docker/login-action@v2
        with:
          username: ${{ secrets.DOCKERHUB_USERNAME }}
          password: ${{ secrets.DOCKERHUB_TOKEN }}

      - name: Build and push
        uses: docker/build-push-action@v4
        with:
          context: .
          push: ${{ github.event_name != 'pull_request' }}
          tags: |
            ${{ secrets.DOCKERHUB_USERNAME }}/agent-api:latest
            ${{ secrets.DOCKERHUB_USERNAME }}/agent-api:${{ github.sha }}
          cache-from: type=gha
          cache-to: type=gha,mode=max
```

## 总结

### 核心要点

1. **Dockerfile**：构建应用镜像
2. **多阶段构建**：优化镜像大小
3. **Docker Compose**：多服务编排
4. **镜像优化**：减小体积、利用缓存
5. **CI/CD**：自动化构建部署

### 最佳实践

- 使用多阶段构建
- 利用构建缓存
- 设置健康检查
- 使用非root用户
- 配置资源限制
