## 一、API基础概念

### 1.1 什么是API？

**API（Application Programming Interface）** 是应用程序编程接口，是软件系统之间通信的桥梁。

**类比理解**：
- API就像餐厅的服务员
- 你（客户端）告诉服务员（API）你想要什么
- 服务员去厨房（服务器）取餐
- 服务员把食物（数据）带回来给你

**API的作用**：
- 让不同的软件系统能够相互通信
- 隐藏复杂的内部实现，暴露简单的接口
- 实现系统的解耦和模块化

### 1.2 API类型

**1. REST API（最常用）**
- 基于HTTP协议
- 使用标准HTTP方法（GET、POST、PUT、DELETE）
- 数据格式通常是JSON

**2. GraphQL API**
- 更灵活的查询语言
- 客户端可以精确指定需要的数据
- Facebook开发

**3. WebSocket API**
- 支持双向实时通信
- 适用于聊天、实时更新场景

**4. RPC API（如gRPC）**
- 远程过程调用
- 高性能，适合微服务

### 1.3 API调用流程

```
客户端                     API服务器
  |                           |
  | --- 1. 发送请求 ---------> |
  |    (URL + Method + Data)  |
  |                           |
  | <--- 2. 返回响应 -------- |
  |    (Status + Data)        |
  |                           |
```

**调用要素**：
- **URL（端点）**：API的地址
- **Method（方法）**：要执行的操作
- **Headers（请求头）**：附加信息（认证、内容类型等）
- **Body（请求体）**：发送的数据

## 二、HTTP协议基础

### 2.1 HTTP请求方法

| 方法 | 用途 | 示例 |
|------|------|------|
| GET | 获取资源 | 获取用户信息 |
| POST | 创建资源 | 创建新用户 |
| PUT | 更新资源（完整） | 更新用户所有信息 |
| PATCH | 更新资源（部分） | 只更新用户名 |
| DELETE | 删除资源 | 删除用户 |

### 2.2 HTTP状态码

**成功（2xx）**：
- `200 OK` - 请求成功
- `201 Created` - 创建成功
- `204 No Content` - 成功但无返回内容

**客户端错误（4xx）**：
- `400 Bad Request` - 请求格式错误
- `401 Unauthorized` - 未认证
- `403 Forbidden` - 无权限
- `404 Not Found` - 资源不存在
- `429 Too Many Requests` - 请求过多（限流）

**服务器错误（5xx）**：
- `500 Internal Server Error` - 服务器内部错误
- `502 Bad Gateway` - 网关错误
- `503 Service Unavailable` - 服务不可用

### 2.3 HTTP请求头

常用的请求头：

```http
Content-Type: application/json
Authorization: Bearer sk-xxxxx
Accept: application/json
User-Agent: MyApp/1.0
```

**重要请求头说明**：
- **Content-Type**：请求体的数据格式
  - `application/json` - JSON格式
  - `application/x-www-form-urlencoded` - 表单格式
  - `multipart/form-data` - 文件上传

- **Authorization**：认证信息
  - `Bearer <token>` - OAuth 2.0令牌
  - `Basic <credentials>` - 基础认证

## 三、REST API设计原则

### 3.1 RESTful API规范

**资源命名**：
- 使用名词，不用动词
- 使用复数形式
- 层级关系清晰

```
好的设计：
GET    /users              # 获取用户列表
GET    /users/123          # 获取ID为123的用户
POST   /users              # 创建新用户
PUT    /users/123          # 更新ID为123的用户
DELETE /users/123          # 删除ID为123的用户
GET    /users/123/orders   # 获取用户的订单列表

不好的设计：
GET    /getUsers           # 不推荐：使用了动词
GET    /user               # 不推荐：应该用复数
```

### 3.2 API版本管理

```
# URL路径方式
/api/v1/users
/api/v2/users

# 请求头方式
Accept: application/vnd.myapi.v1+json
```

### 3.3 分页和过滤

```
# 分页
GET /users?page=1&limit=10

# 过滤
GET /users?status=active&role=admin

# 排序
GET /users?sort=name&order=asc

# 搜索
GET /users?search=john
```

## 四、JSON数据格式详解

### 4.1 JSON基础

**JSON（JavaScript Object Notation）** 是轻量级的数据交换格式。

**基本类型**：
```json
{
  "string": "Hello",
  "number": 42,
  "float": 3.14,
  "boolean": true,
  "null": null,
  "array": [1, 2, 3],
  "object": {
    "key": "value"
  }
}
```

### 4.2 JSON vs Python字典

```python
# Python字典
python_dict = {
    "name": "Alice",
    "age": 25,
    "skills": ["Python", "AI"]
}

# 转换为JSON字符串
import json
json_string = json.dumps(python_dict)

# 解析JSON字符串
parsed = json.loads(json_string)
```

### 4.3 JSON vs JavaScript对象

```javascript
// JavaScript对象（键可以不用引号）
const jsObj = {
  name: "Alice",
  age: 25
};

// JSON字符串（键必须用双引号）
const jsonString = '{"name": "Alice", "age": 25}';

// 解析JSON
const parsed = JSON.parse(jsonString);

// 转换为JSON字符串
const str = JSON.stringify(jsObj);
```

### 4.4 JSON在AI API中的应用

**请求示例**：

## 五、版本控制基础（Git）

### 5.1 Git核心概念

**Git** 是分布式版本控制系统，用于追踪代码变更。

**核心概念**：
- **Repository（仓库）**：存储代码和变更历史
- **Commit（提交）**：保存代码变更的快照
- **Branch（分支）**：独立的开发线
- **Merge（合并）**：将分支合并到主线
- **Remote（远程）**：远程仓库（如GitHub）

### 5.2 Git工作流程

```
工作区 --- add ---> 暂存区 --- commit ---> 本地仓库 --- push ---> 远程仓库
  ^                                                              |
  | <---------------------- pull/clone --------------------------
```

### 5.3 常用Git命令

**初始化和克隆**：
```bash
# 初始化仓库
git init

# 克隆远程仓库
git clone <url>
```

**基本操作**：
```bash
# 查看状态
git status

# 添加文件到暂存区
git add filename        # 添加单个文件
git add .              # 添加所有文件

# 提交变更
git commit -m "提交说明"

# 查看历史
git log
git log --oneline      # 简洁模式
```

**分支操作**：
```bash
# 查看分支
git branch

# 创建分支
git branch feature-name

# 切换分支
git checkout feature-name
git checkout -b feature-name  # 创建并切换

# 合并分支
git merge feature-name

# 删除分支
git branch -d feature-name
```

**远程操作**：
```bash
# 添加远程仓库
git remote add origin <url>

# 推送到远程
git push origin main

# 从远程拉取
git pull origin main

# 查看远程仓库
git remote -v
```

### 5.4 .gitignore文件

用于指定不需要版本控制的文件。

```gitignore
# Python

venv/

# Node.js
node_modules/
```

## 六、包管理

### 6.1 Python包管理（pip）

**安装包**：
```bash
# 安装包
pip install package-name

# 安装特定版本
pip install package-name==1.2.3

# 从requirements.txt安装
pip install -r requirements.txt
```

**requirements.txt**：
```txt
openai>=1.0.0
anthropic>=0.18.0
python-dotenv>=1.0.0
requests>=2.28.0
```

**虚拟环境**：
```bash
# 创建虚拟环境
python -m venv venv

# 激活虚拟环境
source venv/bin/activate  # macOS/Linux
venv\Scripts\activate     # Windows

# 导出依赖
pip freeze > requirements.txt
```

### 6.2 Node.js包管理（npm）

**安装包**：
```bash
# 初始化项目
npm init -y

# 安装生产依赖
npm install package-name

# 安装开发依赖
npm install --save-dev package-name

# 全局安装
npm install -g package-name
```

**package.json**：
```json
{
  "dependencies": {
    "openai": "^4.0.0",
    "dotenv": "^16.0.0"
  },
  "devDependencies": {
    "jest": "^29.0.0"
  }
}
```

**常用命令**：
```bash
npm install          # 安装所有依赖
npm update           # 更新依赖
npm uninstall name   # 卸载包
npm run script-name  # 运行脚本
```

## 七、环境变量管理

### 7.1 为什么需要环境变量？

**用途**：
- 存储敏感信息（API密钥、密码）
- 区分开发环境和生产环境
- 配置信息不硬编码在代码中

### 7.2 .env文件

```
# .env文件（不要提交到Git！）
OPENAI_API_KEY=sk-your-openai-key
ANTHROPIC_API_KEY=sk-ant-your-key
DATABASE_URL=postgresql://localhost/mydb
DEBUG=true
```

### 7.3 在代码中使用

**Python**：
```python
from dotenv import load_dotenv
import os

load_dotenv()  # 加载.env文件

api_key = os.getenv("OPENAI_API_KEY")
```

**JavaScript**：
```javascript
import dotenv from 'dotenv';
dotenv.config();

const apiKey = process.env.OPENAI_API_KEY;
```

### 7.4 最佳实践

- ✅ 使用.env文件存储敏感信息
- ✅ 将.env添加到.gitignore
- ✅ 提供.env.example作为模板
- ❌ 不要将.env文件提交到代码仓库
- ❌ 不要在代码中硬编码API密钥
