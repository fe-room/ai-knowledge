# 1.3.3 调试与测试基础

> 掌握调试技巧和单元测试，编写高质量代码

## 一、单元测试概念

### 1.1 为什么需要测试？

**测试的好处**：
- 发现Bug更早
- 重构代码更安全
- 文档化代码行为
- 提高代码质量

### 1.2 测试类型

```
测试金字塔
    ┌─────────┐
    │  E2E测试  │  （端到端）
    ├─────────┤
    │ 集成测试  │  （多模块协作）
    ├─────────┤
    │ 单元测试  │  （最小单元）
    └─────────┘
```

**单元测试**：测试单个函数或类
**集成测试**：测试多个组件的协作
**E2E测试**：测试完整的用户流程

## 二、Python测试（pytest）

### 2.1 安装pytest

```bash
pip install pytest pytest-cov
```

### 2.2 基本测试

**测试文件命名规则**：
- 文件名以`test_`开头
- 函数名以`test_`开头

**示例：测试计算器函数**

```python
# src/calculator.py
def add(a, b):
    return a + b

def subtract(a, b):
    return a - b

def multiply(a, b):
    return a * b

def divide(a, b):
    if b == 0:
        raise ValueError("除数不能为0")
    return a / b
```

```python
# tests/test_calculator.py
import pytest
from src.calculator import add, subtract, multiply, divide

class TestCalculator:
    def test_add(self):
        assert add(1, 2) == 3
        assert add(-1, 1) == 0
        assert add(0, 0) == 0

    def test_subtract(self):
        assert subtract(5, 3) == 2
        assert subtract(1, 1) == 0

    def test_multiply(self):
        assert multiply(2, 3) == 6
        assert multiply(-1, 3) == -3

    def test_divide(self):
        assert divide(6, 2) == 3

    def test_divide_by_zero(self):
        with pytest.raises(ValueError):
            divide(1, 0)
```

### 2.3 运行测试

```bash
# 运行所有测试
pytest

# 运行指定文件
pytest tests/test_calculator.py

# 运行指定测试
pytest tests/test_calculator.py::TestCalculator::test_add

# 显示详细输出
pytest -v

# 显示print输出
pytest -s
```

### 2.4 测试LLM API调用

**使用Mock模拟API**：

```python
# tests/test_llm_client.py
import pytest
from unittest.mock import Mock, patch
from src.llm_client import LLMClient

class TestLLMClient:
    @patch('src.llm_client.OpenAI')
    def test_chat(self, mock_openai):
        # 设置模拟响应
        mock_response = Mock()
        mock_response.choices = [
            Mock(message=Mock(content="测试回复"))
        ]

        mock_client = Mock()
        mock_client.chat.completions.create.return_value = mock_response
        mock_openai.return_value = mock_client

        # 测试
        client = LLMClient()
        result = client.chat("你好")

        assert result == "测试回复"
```

### 2.5 pytest常用功能

**参数化测试**：
```python
@pytest.mark.parametrize("a,b,expected", [
    (1, 2, 3),
    (0, 0, 0),
    (-1, 1, 0),
])
def test_add_parametrize(a, b, expected):
    assert add(a, b) == expected
```

**fixture（测试夹具）**：
```python
@pytest.fixture
def sample_data():
    return {"name": "Alice", "age": 25}

def test_with_fixture(sample_data):
    assert sample_data["name"] == "Alice"
```

**跳过测试**：
```python
@pytest.mark.skip(reason="暂时跳过")
def test_something():
    pass
```

## 三、JavaScript测试（Jest）

### 3.1 安装Jest

```bash
npm install --save-dev jest
```

### 3.2 基本测试

```javascript
// src/calculator.js
function add(a, b) {
  return a + b;
}

function subtract(a, b) {
  return a - b;
}

module.exports = { add, subtract };
```

```javascript
// tests/calculator.test.js
const { add, subtract } = require('../src/calculator');

describe('Calculator', () => {
  test('adds 1 + 2 to equal 3', () => {
    expect(add(1, 2)).toBe(3);
  });

  test('subtracts 5 - 3 to equal 2', () => {
    expect(subtract(5, 3)).toBe(2);
  });
});
```

### 3.3 运行测试

```bash
# 运行所有测试
npx jest

# 监听模式
npx jest --watch

# 覆盖率报告
npx jest --coverage
```

### 3.4 Jest常用匹配器

```javascript
test('common matchers', () => {
  // 相等
  expect(1 + 1).toBe(2);
  expect({ a: 1 }).toEqual({ a: 1 });

  // 真值
  expect(true).toBeTruthy();
  expect(false).toBeFalsy();
  expect(null).toBeNull();
  expect(undefined).toBeUndefined();

  // 数字
  expect(5).toBeGreaterThan(3);
  expect(5).toBeLessThan(10);

  // 字符串
  expect('hello world').toContain('world');
  expect('hello').toMatch(/ell/);

  // 数组
  expect([1, 2, 3]).toContain(2);

  // 异常
  expect(() => throwError()).toThrow();
});
```

### 3.5 测试异步代码

```javascript
// 测试Promise
test('async test', async () => {
  const data = await fetchData();
  expect(data).toBe('some data');
});

// 测试回调
test('callback test', (done) => {
  fetchDataCallback((data) => {
    expect(data).toBe('some data');
    done();
  });
});
```

## 四、调试技巧

### 4.1 Print调试法

**Python**：
```python
def process_data(data):
    print(f"输入数据: {data}")  # 调试输出
    result = transform(data)
    print(f"处理后: {result}")   # 调试输出
    return result
```

**JavaScript**：
```javascript
function processData(data) {
  console.log('输入数据:', data);
  const result = transform(data);
  console.log('处理后:', result);
  return result;
}
```

### 4.2 日志记录（更专业的方式）

**Python logging**：
```python
import logging

# 配置日志
logging.basicConfig(
    level=logging.DEBUG,
    format='%(asctime)s - %(name)s - %(levelname)s - %(message)s'
)

logger = logging.getLogger(__name__)

def process_data(data):
    logger.debug(f"输入数据: {data}")
    result = transform(data)
    logger.info(f"处理完成: {result}")
    return result
```

**JavaScript console**：
```javascript
console.log('普通日志');
console.warn('警告');
console.error('错误');
console.time('计时');
// ... 代码 ...
console.timeEnd('计时');
```

### 4.3 IDE断点调试

**VS Code调试步骤**：

1. 设置断点：点击行号左侧
2. 启动调试：F5或调试面板
3. 调试控制：
   - F5：继续
   - F10：单步跳过
   - F11：单步进入
   - Shift+F11：单步跳出

**launch.json配置（Python）**：

### 4.4 调试AI应用的特殊技巧

**1. 检查输入输出**：
```python
def call_llm(prompt):
    logger.debug(f"Prompt: {prompt}")
    response = client.chat.completions.create(
        model="gpt-3.5-turbo",
        messages=[{"role": "user", "content": prompt}]
    )
    reply = response.choices[0].message.content
    logger.debug(f"Response: {reply}")
    logger.info(f"Tokens used: {response.usage.total_tokens}")
    return reply
```

**2. 记录Token使用**：
```python
class TokenTracker:
    def __init__(self):
        self.total_tokens = 0

    def track(self, response):
        self.total_tokens += response.usage.total_tokens
        logger.info(f"本次: {response.usage.total_tokens}, 总计: {self.total_tokens}")

tracker = TokenTracker()
```

**3. 测试不同模型**：
```python
@pytest.mark.parametrize("model", [
    "gpt-3.5-turbo",
    "gpt-4",
    "claude-3-5-sonnet-20241022"
])
def test_models(model):
    result = call_llm("你好", model=model)
    assert result is not None
```

## 五、测试覆盖率和CI/CD

### 5.1 测试覆盖率

**Python pytest-cov**：
```bash
# 运行测试并生成覆盖率报告
pytest --cov=src tests/

# 生成HTML报告
pytest --cov=src --cov-report=html tests/
```

**JavaScript Jest**：
```bash
npx jest --coverage
```

### 5.2 持续集成（CI）

**GitHub Actions示例**：
```yaml
# .github/workflows/test.yml
name: Tests

on: [push, pull_request]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3

      - name: Set up Python
        uses: actions/setup-python@v4
        with:
          python-version: '3.10'

      - name: Install dependencies
        run: |
          pip install -r requirements.txt
          pip install pytest pytest-cov

      - name: Run tests
        run: pytest --cov=src tests/
```

## 六、调试AI应用的常见问题

### 问题1：API调用无响应

**调试步骤**：
1. 检查网络连接
2. 检查API密钥是否有效
3. 检查请求参数是否正确
4. 添加超时设置

### 问题2：Token超限

**调试步骤**：
1. 打印输入长度
2. 检查是否需要分块处理
3. 考虑使用更大上下文的模型

### 问题3：输出质量不佳

**调试步骤**：
1. 检查提示词是否清晰
2. 尝试不同的temperature
3. 提供更多上下文
4. 使用更强大的模型
