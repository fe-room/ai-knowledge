## 二、Python开发环境

### 2.1 Python安装

**推荐版本**：Python 3.10+

**安装方式**：

**macOS**：
```bash
# 使用Homebrew
brew install python@3.10

# 或使用pyenv管理多版本
brew install pyenv
pyenv install 3.10.0
pyenv global 3.10.0
```

**Windows**：
```bash
# 从python.org下载安装包
# 或使用chocolatey
choco install python
```

**Linux**：
```bash
# Ubuntu/Debian
sudo apt update
sudo apt install python3.10 python3-pip
```

### 2.2 虚拟环境管理

**为什么需要虚拟环境？**
- 隔离项目依赖
- 避免版本冲突
- 便于部署

**方式一：venv（Python内置）**

```bash
# 创建虚拟环境
python -m venv venv

# 激活虚拟环境
# macOS/Linux
source venv/bin/activate

# Windows
venv\Scripts\activate

# 退出虚拟环境
deactivate
```

**方式二：conda（推荐用于AI开发）**

```bash
# 安装Miniconda
# 下载: https://docs.conda.io/en/latest/miniconda.html

# 创建环境
conda create -n ai-dev python=3.10

# 激活环境
conda activate ai-dev

# 退出环境
conda deactivate

# 查看所有环境
conda env list

# 删除环境
conda env remove -n ai-dev
```

### 2.3 pip包管理

**基本命令**：
```bash
# 安装包
pip install package-name

# 安装特定版本
pip install package-name==1.2.3

# 从requirements.txt安装
pip install -r requirements.txt

# 导出依赖
pip freeze > requirements.txt

# 升级包
pip install --upgrade package-name

# 卸载包
pip uninstall package-name
```

**requirements.txt示例**：
```txt
openai>=1.0.0
anthropic>=0.18.0
python-dotenv>=1.0.0
requests>=2.28.0
aiohttp>=3.8.0
```

## 四、API密钥安全管理（重点）

### 4.1 为什么密钥安全很重要？

**硬编码密钥的风险**：
- 代码泄露时密钥暴露
- GitHub会自动扫描并警告
- 可能导致巨额账单
- 违反服务商使用条款

### 4.2 使用.env文件管理密钥

**步骤1：创建.env文件**

```bash
# 项目根目录
touch .env
```

**步骤2：添加密钥到.env**

```env
# OpenAI API密钥
OPENAI_API_KEY=sk-your-openai-api-key-here

# Anthropic API密钥
ANTHROPIC_API_KEY=sk-ant-your-anthropic-key-here

# 其他配置
DEBUG=true
MODEL_NAME=gpt-3.5-turbo
```

**步骤3：添加.env到.gitignore**

```gitignore
# .gitignore
.env
.env.local
.env.*.local

# 其他敏感文件
*.pem
*.key
secrets/
```

**步骤4：创建.env.example模板**

```env
# .env.example（提交到Git）
OPENAI_API_KEY=your-openai-api-key
ANTHROPIC_API_KEY=your-anthropic-api-key
DEBUG=false
MODEL_NAME=gpt-3.5-turbo
```

### 4.3 在代码中使用环境变量

**Python方式**：

```python
# 安装python-dotenv
# pip install python-dotenv

from dotenv import load_dotenv
import os

# 加载.env文件
load_dotenv()

# 读取环境变量
api_key = os.getenv("OPENAI_API_KEY")

if not api_key:
    raise ValueError("OPENAI_API_KEY未设置")

# 使用密钥
from openai import OpenAI
client = OpenAI(api_key=api_key)
```

**JavaScript方式**：

```javascript
// 安装dotenv
// npm install dotenv

import dotenv from 'dotenv';
dotenv.config();

// 读取环境变量
const apiKey = process.env.OPENAI_API_KEY;

if (!apiKey) {
  throw new Error('OPENAI_API_KEY未设置');
}

// 使用密钥
import OpenAI from 'openai';
const client = new OpenAI({ apiKey });
```

### 4.4 密钥最佳实践

**✅ 应该做的**：
- 使用.env文件存储密钥
- 将.env添加到.gitignore
- 提供.env.example作为模板
- 定期轮换密钥
- 使用最小权限原则
- 监控API使用量

**❌ 不应该做的**：
- 在代码中硬编码密钥
- 将.env文件提交到Git
- 在聊天/邮件中分享密钥
- 使用公共仓库存储密钥
- 给密钥设置过多权限

### 4.5 密钥轮换策略

**定期轮换**：
```bash
# 1. 在服务商平台生成新密钥
# 2. 更新.env文件
# 3. 验证新密钥可用
# 4. 删除旧密钥
```

**紧急情况处理**：
```bash
# 如果密钥泄露：
# 1. 立即在服务商平台删除/禁用密钥
# 2. 生成新密钥
# 3. 更新所有使用该密钥的地方
# 4. 检查账单是否有异常
```

## 六、验证环境配置

### 6.1 Python环境验证

创建测试文件`test_env.py`：

```python
import sys
from dotenv import load_dotenv
import os

def check_environment():
    print("环境检查...")

    # 检查Python版本
    print(f"Python版本: {sys.version}")

    # 检查.env文件
    load_dotenv()

    # 检查API密钥
    openai_key = os.getenv("OPENAI_API_KEY")
    if openai_key:
        print(f"✓ OpenAI API密钥已设置: {openai_key[:10]}...")
    else:
        print("✗ OpenAI API密钥未设置")

    # 检查依赖
    try:
        import openai
        print(f"✓ openai包已安装: {openai.__version__}")
    except ImportError:
        print("✗ openai包未安装")

if __name__ == "__main__":
    check_environment()
```

运行：`python test_env.py`

### 6.2 Node.js环境验证

创建测试文件`test_env.js`：

```javascript
import dotenv from 'dotenv';
dotenv.config();

function checkEnvironment() {
  console.log('环境检查...');

  // 检查Node.js版本
  console.log(`Node.js版本: ${process.version}`);

  // 检查API密钥
  const openaiKey = process.env.OPENAI_API_KEY;
  if (openaiKey) {
    console.log(`✓ OpenAI API密钥已设置: ${openaiKey.substring(0, 10)}...`);
  } else {
    console.log('✗ OpenAI API密钥未设置');
  }
}

checkEnvironment();
```

运行：`node test_env.js`

## 七、常见问题

### Q1: 如何在VS Code中配置Python解释器？

**解答**：
1. `Cmd+Shift+P`（macOS）或`Ctrl+Shift+P`（Windows）
2. 输入`Python: Select Interpreter`
3. 选择虚拟环境中的Python

### Q2: .env文件没有被加载怎么办？

**解答**：
- 确保.env文件在项目根目录
- 确保在代码开头调用`load_dotenv()`
- 检查.env文件格式（无空格、无引号）

### Q3: 如何管理多个项目的不同密钥？

**解答**：
- 每个项目独立.env文件
- 使用不同的密钥名称区分
- 或使用密钥管理服务（如AWS Secrets Manager）
