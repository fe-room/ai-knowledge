# 2.4.1 RAG基础概念

> 理解检索增强生成的原理和应用

## 一、什么是RAG？

### 1.1 定义

**RAG（Retrieval-Augmented Generation）** 是一种结合信息检索和文本生成的技术。

**核心思想**：在生成回答之前，先从知识库中检索相关信息，然后将检索结果作为上下文提供给LLM。

### 1.2 工作原理

├────────────────────────────────────────────────────┤
│                                                    │
│  用户问题                                          │

│      ↓                                             │
│  ┌─────────────┐                                   │

│         │                                          │
│         ↓                                          │
│  ┌─────────────┐      ┌─────────────┐             │

### 1.3 与传统LLM对比

| 特性 | 传统LLM | RAG |
|------|---------|-----|
| 知识来源 | 训练数据 | 外部知识库 |
| 知识时效 | 有截止日期 | 可实时更新 |
| 事实准确性 | 可能幻觉 | 基于检索文档 |
| 可追溯性 | 无法追溯 | 可引用来源 |
| 私有数据 | 无法使用 | 支持私有数据 |
| 成本 | 低（推理） | 中（向量存储） |

## 二、RAG vs 微调

### 2.1 技术对比

```
微调（Fine-tuning）：
┌─────────────────────────────────────┐
│  训练数据 → 模型微调 → 新模型        │
│                                     │
│  特点：                             │
│  - 改变模型参数                     │
│  - 知识内化到模型中                 │
│  - 需要大量训练数据                 │
│  - 更新成本高                       │
└─────────────────────────────────────┘

RAG：
┌─────────────────────────────────────┐
│  外部知识库 + 检索 + 生成            │
│                                     │
│  特点：                             │
│  - 不改变模型参数                   │
│  - 知识存储在外部                   │
│  - 只需维护知识库                   │
│  - 更新成本低                       │
└─────────────────────────────────────┘
```

### 2.2 选择建议

**选择RAG**：
- ✅ 知识频繁更新
- ✅ 需要引用来源
- ✅ 私有数据场景
- ✅ 预算有限
- ✅ 需要解释性

**选择微调**：
- ✅ 特定任务优化
- ✅ 风格适配
- ✅ 知识相对稳定
- ✅ 需要极致性能

**最佳实践**：RAG + 微调结合

## 五、简单RAG实现

```python
"""
简单RAG系统实现
"""

from openai import OpenAI
import numpy as np

class SimpleRAG:
    def __init__(self):
        self.client = OpenAI()
        self.documents = []
        self.embeddings = []

    def add_documents(self, documents):
        """添加文档"""
        for doc in documents:
            # 获取向量
            embedding = self.get_embedding(doc)
            self.documents.append(doc)
            self.embeddings.append(embedding)

    def get_embedding(self, text):
        """获取向量"""
        response = self.client.embeddings.create(
            model="text-embedding-3-small",
            input=text
        )
        return response.data[0].embedding

    def cosine_similarity(self, a, b):
        """余弦相似度"""
        return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))

    def retrieve(self, query, k=3):
        """检索"""
        query_embedding = self.get_embedding(query)

        # 计算相似度
        similarities = [
            self.cosine_similarity(query_embedding, doc_emb)
            for doc_emb in self.embeddings
        ]

        # 获取top-k
        top_indices = np.argsort(similarities)[-k:][::-1]

        return [self.documents[i] for i in top_indices]

    def generate(self, query, context):
        """生成回答"""
        prompt = f"""
        基于以下上下文回答问题：

        上下文：
        {context}

        问题：{query}

        请基于上下文回答，如果上下文中没有相关信息，请说明。
        """

        response = self.client.chat.completions.create(
            model="gpt-3.5-turbo",
            messages=[{"role": "user", "content": prompt}]
        )

        return response.choices[0].message.content

    def query(self, question):
        """查询"""
        # 检索
        relevant_docs = self.retrieve(question)

        # 构建上下文
        context = "\n\n".join(relevant_docs)

        # 生成
        answer = self.generate(question, context)

        return {
            "answer": answer,
            "sources": relevant_docs
        }

# 使用示例
rag = SimpleRAG()

# 添加文档
documents = [
    "Python是一种解释型编程语言，由Guido van Rossum创建。",
    "RAG代表检索增强生成，是一种结合检索和生成的技术。",
    "向量数据库用于存储和检索高维向量数据。"
]
rag.add_documents(documents)

# 查询
result = rag.query("什么是RAG？")
print(result["answer"])
print("\n来源文档:", result["sources"])
```

## 六、RAG的挑战

### 6.1 常见问题

**1. 检索质量**
- 问题：检索不到相关文档
- 解决：优化分块策略、使用混合检索

**2. 上下文长度**
- 问题：检索内容超过模型上下文限制
- 解决：智能截断、压缩、重排序

**3. 答案质量**
- 问题：LLM未正确利用检索内容
- 解决：优化提示词、使用更强的模型

**4. 实时更新**
- 问题：知识库更新后索引未更新
- 解决：增量索引、定时同步

## 总结

### 核心要点

1. **RAG** = 检索 + 生成，结合外部知识库
2. **优势**：知识可更新、可追溯、支持私有数据
3. **架构**：索引阶段 + 查询阶段
4. **核心组件**：文档处理、向量存储、检索、生成
