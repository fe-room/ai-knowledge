## 二、框架详细对比

### 2.1 LangChain

```python
"""
LangChain特点
"""

优点：
+ 组件丰富（Models、Prompts、Chains等）
+ 社区活跃，文档完善
+ 链式组合简单易懂
+ 与各种LLM集成良好
+ RAG支持完善

缺点：
- 状态管理较弱
- 循环执行需要特殊处理
- 复杂Agent难以实现
- 版本更新频繁，API不稳定

适用场景：
✅ 简单问答系统
✅ RAG应用
✅ 单一流程任务
✅ 初学者入门
```

### 2.2 LangGraph

```python
"""
LangGraph特点
"""

优点：
+ 强大的状态管理
+ 支持循环和分支
+ 可持久化执行状态
+ 复杂Agent实现
+ 可视化图结构

缺点：
- 学习曲线陡峭
- 需要理解图概念
- 相对较新，文档较少
- 调试复杂

适用场景：
✅ 复杂Agent应用
✅ 需要循环执行的任务
✅ ReAct、Plan-Execute模式
✅ 多步骤有状态任务
```

### 2.3 AutoGen

```python
"""
AutoGen特点（微软）
"""

优点：
+ 多Agent协作设计
+ 人机交互支持
+ 代码执行环境
+ 灵活的对话模式

缺点：
- 专注于多Agent
- 配置复杂
- 资源消耗较大
- 主要用于研究

适用场景：
✅ 多Agent协作系统
✅ 需要人参与的对话
✅ 代码生成和执行
✅ 研究实验
```

### 2.4 CrewAI

```python
"""
CrewAI特点
"""

优点：
+ 团队协作概念清晰
+ 任务分工明确
+ 易于定义角色
+ 简洁的API

缺点：
- 功能相对有限
- 社区较小
- 不适合复杂状态

适用场景：
✅ 任务分工明确的项目
✅ 角色扮演场景
✅ 团队模拟
```

### 3.2 选择指南

```python
"""
框架选择指南
"""

def choose_framework(requirements):
    """根据需求选择框架"""

    # 判断因素
    factors = {
        "单流程": {"langchain": 5, "langgraph": 3},
        "多步骤循环": {"langchain": 2, "langgraph": 5},
        "多Agent": {"autogen": 5, "crewai": 4},
        "RAG": {"langchain": 5, "haystack": 4},
        "状态管理": {"langgraph": 5, "langchain": 2},
        "易学性": {"langchain": 5, "langgraph": 3},
        "企业集成": {"semantic_kernel": 5, "langchain": 3}
    }

    # 计算得分
    scores = {}
    for req, weight in requirements.items():
        for framework, score in factors.get(req, {}).items():
            scores[framework] = scores.get(framework, 0) + score * weight

    # 推荐最高得分
    best = max(scores, key=scores.get)
    return best

# 使用示例
requirements = {
    "单流程": 1,
    "RAG": 1,
    "易学性": 1
}
print(choose_framework(requirements))  # 输出: langchain
```

## 四、组合使用

### 4.1 LangChain + LangGraph

```python
"""
LangChain和LangGraph组合使用
"""

from langchain_openai import ChatOpenAI
from langchain.prompts import ChatPromptTemplate
from langgraph.graph import StateGraph, END

# 使用LangChain的LLM和Prompts
llm = ChatOpenAI()
prompt = ChatPromptTemplate.from_template("处理: {input}")

# 使用LangGraph的状态管理
class State(TypedDict):
    input: str
    output: str

def process_node(state: State) -> State:
    # 使用LangChain组件
    chain = prompt | llm
    result = chain.invoke({"input": state["input"]})
    state["output"] = result.content
    return state

# 创建LangGraph图
workflow = StateGraph(State)
workflow.add_node("process", process_node)
workflow.add_edge("process", END)
workflow.set_entry_point("process")

app = workflow.compile()
```

## 总结

### 核心要点

1. **框架多样**：LangChain、LangGraph、AutoGen、CrewAI等
2. **各有侧重**：链式、图式、多Agent、团队协作
3. **选择依据**：任务复杂度、学习难度、社区支持
4. **组合使用**：可以组合多个框架的优势
5. **学习路径**：从简单到复杂逐步学习

### 选择建议

| 任务类型 | 推荐框架 |
|---------|---------|
| 简单问答 | LangChain |
| RAG应用 | LangChain/Haystack |
| 复杂Agent | LangGraph |
| 多Agent | AutoGen/CrewAI |
| 企业集成 | Semantic Kernel |
