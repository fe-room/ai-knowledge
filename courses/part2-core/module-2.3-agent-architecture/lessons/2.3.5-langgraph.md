## 一、LangGraph概述

### 1.1 什么是LangGraph？

**LangGraph** 是LangChain的扩展框架，用于构建有状态的、循环的Agent应用。

**核心特点**：
- 状态管理
- 循环图结构
- 条件分支
- 持久化执行

```
LangChain vs LangGraph：

LangChain（链式）：
A → B → C → D

LangGraph（图式）：
    ┌───────┐
    │  Start │
    └───┬───┘
        ↓
    ┌───────┐
    │  Node  │←──┐
    └───┬───┘   │
        ↓       │
    ┌───────┐   │
    │ Decision ││
    └───┬───┘   │
    Yes │   No  │
        ↓       │
    ┌───────┐   │
    │  Node  │──┘
    └───┬───┘
        ↓
    ┌───────┐
    │  End   │
    └───────┘
```

## 二、核心概念

### 2.1 StateGraph

```python
"""
状态图基础
"""

from langgraph.graph import StateGraph, END
from typing import TypedDict, Annotated

# 定义状态
class AgentState(TypedDict):
    messages: list
    current_step: str
    result: str

# 创建图
workflow = StateGraph(AgentState)
```

### 2.2 Nodes和Edges

```python
"""
节点和边
"""

# 定义节点函数
def node_a(state: AgentState) -> AgentState:
    """节点A"""
    state["current_step"] = "node_a"
    state["messages"].append("执行节点A")
    return state

def node_b(state: AgentState) -> AgentState:
    """节点B"""
    state["current_step"] = "node_b"
    state["messages"].append("执行节点B")
    return state

# 添加节点
workflow.add_node("node_a", node_a)
workflow.add_node("node_b", node_b)

# 添加边
workflow.add_edge("node_a", "node_b")
workflow.add_edge("node_b", END)

# 设置入口
workflow.set_entry_point("node_a")

# 编译
app = workflow.compile()
```

## 三、条件分支

### 3.1 条件路由

```python
"""
条件分支
"""

def router(state: AgentState) -> str:
    """路由决策"""
    if len(state["messages"]) > 5:
        return "finish"
    else:
        return "continue"

def process(state: AgentState) -> AgentState:
    """处理节点"""
    state["messages"].append(f"处理步骤{len(state['messages'])}")
    return state

def finish(state: AgentState) -> AgentState:
    """完成节点"""
    state["result"] = "完成"
    return state

# 创建图
workflow = StateGraph(AgentState)

workflow.add_node("process", process)
workflow.add_node("finish", finish)

# 条件边
workflow.add_conditional_edges(
    "process",
    router,
    {
        "continue": "process",
        "finish": "finish"
    }
)

workflow.add_edge("finish", END)
workflow.set_entry_point("process")

app = workflow.compile()

# 运行
result = app.invoke({"messages": [], "current_step": "", "result": ""})
print(result)
```

## 四、Agent实现

### 4.1 ReAct Agent

```python
"""
LangGraph实现ReAct Agent
"""

from langgraph.graph import StateGraph, END
from langchain_openai import ChatOpenAI
from langchain.tools import Tool
import json

class ReActState(TypedDict):
    question: str
    thoughts: list
    actions: list
    observations: list
    answer: str

def think(state: ReActState) -> ReActState:
    """思考节点"""
    llm = ChatOpenAI()

    history = "\n".join([
        f"Thought: {t}\nAction: {a}\nObservation: {o}"
        for t, a, o in zip(
            state["thoughts"],
            state["actions"],
            state["observations"]
        )
    ])

    prompt = f"""
问题：{state["question"]}
历史：{history}

思考下一步。
"""

    response = llm.invoke(prompt)
    thought = response.content
    state["thoughts"].append(thought)

    return state

def act(state: ReActState) -> ReActState:
    """行动节点"""
    # 解析行动
    last_thought = state["thoughts"][-1]

    if "Finish:" in last_thought:
        answer = last_thought.split("Finish:")[-1].strip()
        state["answer"] = answer
        return state

    # 执行工具
    # ... 简化示例
    action = "search"
    action_input = "查询内容"

    result = f"工具结果"
    state["actions"].append(action)
    state["observations"].append(result)

    return state

def should_continue(state: ReActState) -> str:
    """判断是否继续"""
    if state["answer"]:
        return "end"
    if len(state["thoughts"]) > 10:
        return "end"
    return "continue"

# 创建图
workflow = StateGraph(ReActState)

workflow.add_node("think", think)
workflow.add_node("act", act)

workflow.add_edge("think", "act")
workflow.add_conditional_edges(
    "act",
    should_continue,
    {"continue": "think", "end": END}
)

workflow.set_entry_point("think")

app = workflow.compile()
```

## 五、循环执行

### 5.1 循环处理

```python
"""
循环处理任务
"""

class LoopState(TypedDict):
    items: list
    processed: list
    current_index: int

def process_item(state: LoopState) -> LoopState:
    """处理单个项目"""
    items = state["items"]
    index = state["current_index"]

    if index < len(items):
        # 处理当前项
        result = f"处理: {items[index]}"
        state["processed"].append(result)
        state["current_index"] = index + 1

    return state

def should_continue(state: LoopState) -> str:
    """是否继续"""
    if state["current_index"] >= len(state["items"]):
        return "end"
    return "continue"

# 创建循环图
workflow = StateGraph(LoopState)

workflow.add_node("process", process_item)
workflow.add_conditional_edges(
    "process",
    should_continue,
    {"continue": "process", "end": END}
)

workflow.set_entry_point("process")

app = workflow.compile()

# 运行
result = app.invoke({
    "items": ["A", "B", "C", "D"],
    "processed": [],
    "current_index": 0
})

print(result)
```

## 六、并行执行

### 6.1 并行节点

```python
"""
并行执行
"""

class ParallelState(TypedDict):
    input: str
    results: dict

def task_a(state: ParallelState) -> ParallelState:
    """任务A"""
    result = f"A处理: {state['input']}"
    state["results"]["task_a"] = result
    return state

def task_b(state: ParallelState) -> ParallelState:
    """任务B"""
    result = f"B处理: {state['input']}"
    state["results"]["task_b"] = result
    return state

def task_c(state: ParallelState) -> ParallelState:
    """任务C"""
    result = f"C处理: {state['input']}"
    state["results"]["task_c"] = result
    return state

def combine(state: ParallelState) -> ParallelState:
    """合并结果"""
    combined = "\n".join(state["results"].values())
    state["results"]["combined"] = combined
    return state

# 创建并行图
workflow = StateGraph(ParallelState)

workflow.add_node("task_a", task_a)
workflow.add_node("task_b", task_b)
workflow.add_node("task_c", task_c)
workflow.add_node("combine", combine)

# 并行执行
workflow.set_entry_point("task_a")
# 实际并行需要特殊处理
workflow.add_edge("task_a", "combine")
workflow.add_edge("task_b", "combine")
workflow.add_edge("task_c", "combine")
workflow.add_edge("combine", END)
```

## 总结

### 核心要点

1. **LangGraph**：有状态的循环图框架
2. **StateGraph**：状态驱动的图结构
3. **条件分支**：动态路由决策
4. **循环执行**：重复处理任务
5. **并行执行**：多任务并发
6. **持久化**：状态保存和恢复

### 与LangChain对比

| 特性 | LangChain | LangGraph |
|------|-----------|-----------|
| 结构 | 链式 | 图式 |
| 状态 | 弱状态 | 强状态管理 |
| 循环 | 不支持 | 支持 |
| 条件分支 | 有限 | 强大 |
| 适用场景 | 简单流程 | 复杂Agent |
