# 2.1.1 提示词基础

> 掌握提示词设计的核心原则和常见模式

## 一、提示词设计原则

### 1.1 原则概览

**好的提示词应该**：
- ✅ 清晰明确
- ✅ 具体详细
- ✅ 有上下文
- ✅ 有示例
- ✅ 有约束

**避免的问题**：
- ❌ 模糊不清
- ❌ 过于宽泛
- ❌ 缺少上下文
- ❌ 有歧义

### 1.2 核心原则详解

**原则1：清晰明确**

```
❌ 差的提示词：
"写点东西"

✅ 好的提示词：
"写一篇500字的文章，主题是'人工智能对教育的影响'，
面向普通读者，风格轻松有趣。"
```

**原则2：提供上下文**

```
❌ 差的提示词：
"翻译这句话"

✅ 好的提示词：
"我正在准备一个技术演讲，需要将以下中文技术术语
翻译成英文，请使用业界通用的翻译：

术语：机器学习、深度学习、神经网络
```

**原则3：指定输出格式**

```
❌ 差的提示词：
"列出常用的编程语言"

✅ 好的提示词：
"列出5种最流行的编程语言，以JSON格式输出：
{
  "languages": [
    {"name": "语言名", "use_case": "主要用途"}
  ]
}
```

**原则4：设置约束**

```
❌ 差的提示词：
"介绍Python"

✅ 好的提示词：
"用3-5句话简要介绍Python编程语言，不超过100字，
适合初学者阅读。"
```

## 二、提示词结构框架

### 2.1 完整提示词结构

一个完整的提示词通常包含以下要素：

```
┌─────────────────────────────────────┐
│          完整提示词结构              │
├─────────────────────────────────────┤
│ 1. 角色设定（Role）                  │
│    "你是一个专业的Python编程助手..."  │
│                                     │
│ 2. 任务描述（Task）                  │
│    "请帮我写一个函数，实现..."       │
│                                     │
│ 3. 上下文（Context）                 │
│    "背景：我正在开发一个Web应用..."   │
│                                     │
│ 4. 示例（Examples）                  │
│    "例如：输入xxx，输出xxx"          │
│                                     │
│ 5. 输出格式（Format）                │
│    "请以Markdown格式输出..."         │
│                                     │
│ 6. 约束条件（Constraints）           │
│    "注意：代码必须有注释..."         │
└─────────────────────────────────────┘
```

### 2.2 实际示例

**示例1：编程助手**

```
# 角色设定
你是一个资深Python开发工程师，擅长编写高质量、可维护的代码。

# 任务
请编写一个函数，实现从URL下载文件并保存到本地的功能。

# 要求
1. 支持断点续传
2. 显示下载进度
3. 错误处理完善
4. 代码有详细注释

# 输出格式
请提供：
1. 完整代码
2. 使用示例
3. 注意事项说明
```

**示例2：文本分析**

```
# 角色
你是一个专业的文本分析师，擅长情感分析和观点提取。

# 任务
分析以下用户评论的情感倾向，并提取关键观点。

# 输入
"这家餐厅的服务态度很好，菜品也很美味，就是价格有点贵，
不过整体体验还是不错的，下次还会再来。"

# 输出格式
{
  "sentiment": "正面/负面/中性",
  "confidence": 0.85,
  "key_points": [
    {"aspect": "服务", "opinion": "好"},
    {"aspect": "菜品", "opinion": "美味"},
    {"aspect": "价格", "opinion": "贵"}
  ],
  "summary": "一句话总结"
}
```

## 三、常见提示词模式

### 3.1 角色扮演模式

```
你是一个[角色]，具有[特点]。

请[任务描述]。

注意：[约束条件]。
```

**示例**：

```
你是一个资深的软件架构师，有15年的系统设计经验，
擅长高可用、高性能系统的设计。

请为电商平台设计一个订单系统架构。

要求：
1. 支持每秒10000订单
2. 99.99%可用性
3. 详细说明技术选型理由
```

### 3.2 任务分解模式

```
请完成以下任务，按步骤执行：

步骤1：[子任务1]
步骤2：[子任务2]
步骤3：[子任务3]

最后：汇总结果。
```

**示例**：

```
请分析这段代码的性能问题，按以下步骤：

步骤1：识别潜在的性能瓶颈
步骤2：分析每个瓶颈的影响程度
步骤3：提供优化建议

代码：
[代码内容]
```

### 3.3 问答模式

```
问题：[问题内容]

背景信息：
- [背景1]
- [背景2]

请回答：
1. [回答要点1]
2. [回答要点2]
```

### 3.4 比较模式

```
比较[A]和[B]的异同：

请从以下维度比较：
1. [维度1]
2. [维度2]
3. [维度3]

输出为表格形式。
```

### 3.5 转换模式

```
输入：[原始内容]

请将其转换为[目标格式]。

要求：
- [要求1]
- [要求2]
```

## 四、提示词技巧

### 4.1 角色设定技巧

**为什么设定角色？**
- 激活模型特定领域的知识
- 调整输出风格和语气
- 提高回答的专业性

**常用角色设定**：

```
# 编程相关
"你是一个Python专家，擅长编写简洁高效的代码。"
"你是一个前端开发工程师，精通React和Vue。"
"你是一个代码审查专家，擅长发现潜在问题。"

# 写作相关
"你是一个专业文案，擅长吸引人的标题和内容。"
"你是一个技术文档撰写专家，内容清晰易懂。"

# 分析相关
"你是一个数据分析师，擅长从数据中发现洞察。"
"你是一个产品经理，擅长需求分析和产品设计。"

# 教育相关
"你是一个耐心的编程老师，善于用简单的语言解释复杂概念。"
"你是一个学习教练，擅长制定学习计划。"
```

### 4.2 输出控制技巧

**控制长度**：
```
"用3句话回答"
"控制在200字以内"
"详细解释，不少于500字"
```

**控制格式**：
```
"以列表形式输出"
"以JSON格式输出"
"以Markdown表格输出"
"以代码形式输出"
```

**控制风格**：
```
"使用正式的书面语"
"使用轻松幽默的语气"
"使用简单易懂的语言"
```

### 4.3 思维引导技巧

**引导推理**：
```
"让我们一步步分析："
"首先...然后...最后..."
"从以下几个角度思考："
```

**引导创新**：
```
"请提供至少3种不同的方案"
"请想一些创新的想法"
"请跳出常规思维"
```

## 五、常见问题与解决

### 5.1 输出太宽泛

**问题**：
```
提示词：介绍Python
输出：过于宽泛，没有重点
```

**解决**：
```
"请面向初学者，用300字介绍Python的主要特点，
重点说明：
1. 为什么Python适合初学者
2. Python的主要应用领域
3. 学习Python的建议路径"
```

### 5.2 输出格式不符合预期

**问题**：
```
提示词：列出常用的设计模式
输出：格式混乱
```

**解决**：
```
"请列出5种常用的设计模式，以表格形式输出：

| 设计模式 | 类型 | 使用场景 | 示例 |

每种设计模式提供一句话说明。"
```

### 5.3 理解错误

**问题**：
```
提示词：写一个函数处理数据
输出：理解错误，写了不相关的函数
```

**解决**：
```
"背景：我正在开发一个用户管理系统。

任务：写一个Python函数，接收用户数据字典，
验证数据有效性，返回处理结果。

输入示例：{'name': 'Alice', 'age': 25}
输出示例：{'valid': True, 'data': {...}}

函数要求：
1. 验证name非空且是字符串
2. 验证age是正整数
3. 返回验证结果和处理后的数据"
```

## 六、提示词模板库

### 6.1 通用模板

**编程任务模板**：

```
# 角色
你是一个[语言/框架]专家。

# 任务
请编写一个[功能描述]的函数/模块。

# 要求
1. [要求1]
2. [要求2]
3. 代码有注释

# 输入示例
[输入示例]

# 输出示例
[输出示例]
```

**文本生成模板**：

```
# 角色
你是一个专业的内容创作者。

# 任务
请写一篇关于[主题]的文章。

# 要求
- 长度：[字数]
- 风格：[风格]
- 受众：[目标读者]

# 大纲
1. [部分1]
2. [部分2]
3. [部分3]
```

**问题分析模板**：

```
# 背景
[背景信息]

# 问题
[问题描述]

# 已尝试的方法
1. [方法1]
2. [方法2]

# 请帮我
1. 分析问题原因
2. 提供解决方案
3. 给出预防建议
```

## 总结

### 核心要点

1. **清晰明确**：避免模糊表述
2. **完整结构**：角色+任务+上下文+格式+约束
3. **提供示例**：帮助模型理解预期输出
4. **迭代优化**：持续改进提示词
