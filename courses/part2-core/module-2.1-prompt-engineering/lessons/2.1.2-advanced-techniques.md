# 2.1.2 高级提示词技术

> 掌握Zero-shot、Few-shot、CoT、ReAct等高级技术

## 一、Zero-shot与Few-shot

### 1.1 Zero-shot（零样本学习）

**定义**：不提供示例，直接让模型完成任务。

```
提示词：
将以下英文翻译成中文：
"Hello, World!"

输出：
"你好，世界！"
```

**适用场景**：
- 简单任务
- 模型已有足够能力的任务
- 通用任务

### 1.2 Few-shot（少样本学习）

**定义**：提供少量示例，帮助模型理解任务。

```
提示词：
判断以下句子的情感：

句子："这部电影太精彩了！"
情感：正面

句子："服务态度很差，不会再来了。"
情感：负面

句子："今天天气不错，适合出去玩。"
情感：正面

句子："这个产品一般般，没什么特别的。"
情感：
```

**输出**：
```
中性
```

### 1.3 Few-shot的优势

**提高准确性**：
```
❌ Zero-shot:
"提取产品信息：iPhone 15 Pro，价格8999元"
→ 可能格式不统一

✅ Few-shot:
提取产品信息：

产品：MacBook Pro
价格：12999元
输出：{"product": "MacBook Pro", "price": 12999}

产品：iPhone 15 Pro
价格：8999元
输出：
```

**统一输出格式**：
```
请按以下格式总结新闻：

新闻：[新闻1]
总结：标题是XX，主要内容是XX，影响是XX。

新闻：[新闻2]
总结：标题是XX，主要内容是XX，影响是XX。

新闻：[实际新闻]
总结：
```

### 1.4 Few-shot最佳实践

**示例选择原则**：
1. **多样性**：覆盖不同情况
2. **典型性**：选择代表性示例
3. **一致性**：格式统一
4. **适量**：通常3-5个足够

**示例**：
```
提取技术文章的关键信息：

文章标题：Python 3.12新特性详解
文章内容：Python 3.12发布，带来了多项性能优化...
输出：
{
  "topic": "Python",
  "type": "新版本特性",
  "key_points": ["性能优化", "新语法特性", "标准库更新"]
}

文章标题：React 19性能优化指南
文章内容：本文介绍React 19中的性能优化技巧...
输出：
{
  "topic": "React",
  "type": "性能优化",
  "key_points": ["React.memo使用", "虚拟化列表", "代码分割"]
}

文章标题：GPT-5技术报告解读
文章内容：OpenAI发布GPT-5技术报告...
输出：
```

## 二、Chain-of-Thought（思维链）

### 2.1 什么是CoT？

**Chain-of-Thought（思维链）**：让模型一步步推理，展示思考过程。

**核心思想**：通过显式的推理步骤，提高复杂任务的准确性。

### 2.2 CoT示例

**数学问题**：

```
❌ 不使用CoT：
问题：小明有5个苹果，给了小红2个，又买了3个，现在有几个？
答案：6个

✅ 使用CoT：
问题：小明有5个苹果，给了小红2个，又买了3个，现在有几个？

让我们一步步思考：
1. 小明原来有5个苹果
2. 给了小红2个，还剩：5 - 2 = 3个
3. 又买了3个，现在有：3 + 3 = 6个

答案：小明现在有6个苹果。
```

**逻辑推理**：

```
问题：所有的猫都是动物，所有的动物都需要水，猫需要水吗？

让我们一步步推理：
1. 已知条件：
   - 所有的猫都是动物
   - 所有的动物都需要水

2. 推理过程：
   - 因为猫是动物
   - 因为所有动物都需要水
   - 所以猫也需要水

3. 结论：
   是的，猫需要水。
```

### 2.3 CoT触发方式

**方式1：直接指令**
```
请一步步思考并回答：
[问题]
```

**方式2：示例引导**
```
问题：[示例问题]
思考过程：
1. ...
2. ...
答案：...

问题：[实际问题]
思考过程：
```

**方式3：自我提问**
```
在回答之前，请先问自己：
1. 这个问题的关键是什么？
2. 需要哪些信息？
3. 如何一步步解决？

问题：[问题]
```

### 2.4 CoT适用场景

**适合**：
- 数学计算
- 逻辑推理
- 复杂决策
- 多步骤任务

**不适合**：
- 简单问答
- 事实查询
- 创意写作

## 三、ReAct框架

### 3.1 什么是ReAct？

**ReAct = Reasoning + Acting**

结合推理和行动，让模型能够：
- 思考下一步做什么
- 执行动作（如调用工具）
- 观察结果
- 继续思考

### 3.2 ReAct流程

```
┌─────────────────────────────────────────────┐
│                  ReAct循环                   │
├─────────────────────────────────────────────┤
│                                             │
│   Thought（思考）→ Action（行动）→ Observation（观察）│
│         ↑                              │    │
│         └──────────────────────────────┘    │
│                                             │
│   重复直到任务完成                            │
└─────────────────────────────────────────────┘
```

### 3.3 ReAct示例

**任务：查询北京明天的天气，并给出穿衣建议**

```
Thought: 用户想知道北京明天的天气和穿衣建议。
我需要先查询天气信息。

Action: 调用天气API
Parameters: {"city": "北京", "date": "明天"}

Observation: 北京明天多云，温度15-22°C，有30%降雨概率

Thought: 已获取天气信息，现在可以给出穿衣建议。
温度适中但有降雨概率，建议携带雨具。

Action: 生成回答

Final Answer:
北京明天多云，温度15-22°C，有30%的降雨概率。
穿衣建议：
1. 穿长袖或薄外套，温度适中
2. 建议携带雨伞，以防降雨
3. 适合户外活动，但要做好防雨准备
```

### 3.4 ReAct提示词模板

```
你是一个智能助手，可以使用工具完成任务。
请按照以下格式思考和行动：

Question: 用户的问题
Thought: 你的思考过程
Action: 要执行的动作（可选）
Observation: 动作的结果（可选）
...（Thought/Action/Observation可重复多次）
Thought: 最终思考
Final Answer: 最终答案

可用工具：
- search: 搜索信息
- calculator: 计算数学表达式
- weather: 查询天气

开始！

Question: [用户输入]
```

## 四、其他高级技术

### 4.1 Self-Consistency（自我一致性）

**原理**：多次采样，选择最常见的答案。

```
# 伪代码示例
answers = []
for i in range(5):
    answer = model.generate(prompt, temperature=0.7)
    answers.append(answer)

final_answer = most_common(answers)
```

### 4.2 Tree of Thoughts（思维树）

**原理**：探索多个推理路径，选择最优。

```
问题：如何在一个月内学会Python？

思维树：
├── 路径1：自学
│   ├── 优点：自由、成本低
│   └── 缺点：效率低、容易放弃
├── 路径2：参加培训
│   ├── 优点：系统、有指导
│   └── 缺点：成本高
└── 路径3：项目驱动
    ├── 优点：实践性强
    └── 缺点：基础可能不牢

建议：结合路径1和3，自学+小项目实践。
```

### 4.3 Prompt Chaining（提示词链）

**原理**：将复杂任务分解为多个步骤。

```
步骤1：提取关键信息
→ 输入：原始文本
→ 输出：关键信息

步骤2：分析关键信息
→ 输入：关键信息
→ 输出：分析结果

步骤3：生成报告
→ 输入：分析结果
→ 输出：最终报告
```

### 示例2：使用CoT解决数学问题

```python
prompt = """
解决问题，请展示思考过程：

问题：一个长方形的周长是24厘米，长是宽的2倍，求长方形的面积。

思考过程：
1. 设宽为x厘米，则长为2x厘米
2. 周长公式：2(长 + 宽) = 24
3. 代入：2(2x + x) = 24
4. 化简：6x = 24
5. 解得：x = 4（宽）
6. 长 = 2x = 8厘米
7. 面积 = 长 × 宽 = 8 × 4 = 32平方厘米

答案：长方形的面积是32平方厘米。
"""
```

## 六、技术选择指南

| 任务类型 | 推荐技术 | 原因 |
|---------|---------|------|
| 简单分类 | Zero-shot | 模型已有足够能力 |
| 复杂分类 | Few-shot | 示例帮助理解边界 |
| 数学计算 | CoT | 展示推理过程 |
| 逻辑推理 | CoT | 提高准确性 |
| 多步骤任务 | ReAct | 需要调用工具 |
| 创意生成 | 多次采样 | 增加多样性 |

---

## 总结

### 核心要点

1. **Zero-shot**：不提供示例，适合简单任务
2. **Few-shot**：提供示例，提高准确性和一致性
3. **CoT**：展示推理过程，适合复杂任务
4. **ReAct**：推理+行动，适合需要工具的任务
